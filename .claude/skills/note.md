# /note — 読書ノートスキル

ECC を読んで気づいたこと、01_analysis への補足・解釈・応用アイデアを `Output/02_notes/` に記録する。

## 使い方

```
/note <トピック>

例:
  /note agent-routing の設計思想について
  /note hooks と skills の使い分け
  /note fb スキルから学んだフィードバックループの作り方
```

## ノート作成の手順

1. **トピックを確認**: 何について書くかを明確にする
2. **関連する01_analysisを確認**: すでに解析メモがあれば参照する
3. **ノートファイルを作成**: `Output/02_notes/YYYY-MM-DD_{トピック}.md`
4. **自分の言葉で書く**: AI の出力をコピーせず、自分の解釈で記述する

## 出力テンプレート

```markdown
# {トピック}

**日付**: YYYY-MM-DD
**関連解析**: → [01_analysis/{関連ファイル}.md]（あれば）
**関連ECC**: ECC/{ファイルパス}

## 今日気づいたこと
（読んで発見した事実・驚いた点）

## 自分の解釈
（自分なりにどう理解したか）

## 個人開発・仕事への応用
（これをどう使えるか、どう活かすか）

## 次に調べること
（理解を深めるために次にやること）
```

## 書き方のガイドライン

- **「AI の解析」と「自分の考え」を混ぜない**: 解析は 01_analysis、考察はここ
- **断定しすぎない**: 「〜だと思う」「〜かもしれない」でOK。後で修正できる
- **疑問が湧いたら別途 /question で記録**: ノートに疑問を埋め込まない
- **02_notes/README.md のインデックスを更新する**
