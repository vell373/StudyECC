# 02_notes — 読書ノート・考察

ECC を読んで自分が考えたこと、01_analysis への補足・異論・応用アイデアを記録する場所。
「自分の言葉で理解する」ことを目的とする。

## ファイル命名規則

```
YYYY-MM-DD_{トピック}.md

例:
  2026-05-01_agent-routing-thoughts.md
  2026-05-10_why-hooks-matter.md
```

## 書き方の指針

- AI の解析（01_analysis）をそのまま転記しない
- 「自分はこう解釈した」「これは個人開発で使えそう」など**自分の視点**を必ず入れる
- 01_analysis との対応は `→ [01_analysis/xxx.md]` で参照
