## 6. 実装上の工夫

1. **スコアリングの透明性**：YAML / JSON で停止条件とスコア基準を可視化し、開発者の実装を支援
2. **段階的説明**：従来パターン（3 つ） → GAN スタイル（新規）という流れで読者の理解を段階化
3. **図解の活用**：抽象的なエージェント連携を ASCII アートで視覚化
4. **ユースケースの具体化**：各パターンに「具体的なタスク例」を付与して実装イメージを支援
5. **デメリット明記**：GAN スタイルが「銀の弾」ではなく「適切に選べば強力」という公正な視点
