# テーマC: 記事執筆

## タイトル
「マルチエージェントハーネスの設計パターン：GANスタイルで品質を最大化する」

## 背景・目的
Claude Code では複数のエージェントを連携させて複雑なタスクを解決します。
その際の「連携パターン」として、ECC では「GAN スタイル」（Generator-Adversarial-Network）という興味深い設計が採用されています。
このタスクでは、マルチエージェント設計の代表的なパターンを解説し、特に「GAN スタイル」がなぜ有効か、どう実装するかを説明する技術記事を執筆してください。

## 想定読者
- Claude Code を使い始めたエンジニア
- AI システムのアーキテクチャに興味がある人
- マルチエージェント設計を学びたい人

## 要求事項

### 必須要素
- **文字数**: 2000～3000文字（日本語）
- **構成**: 導入 → 従来のパターン → GAN スタイルとは → メリット・デメリット → 実装方法 → まとめ
- **パターン解説**: 最低3つの異なるマルチエージェント設計パターンを比較（例：Pipeline、Collaborative、GAN など）
- **実装例**: GAN スタイル実装のコード例・ファイル構成例を含む

### 出力形式
- Markdown 形式
- パターン比較表を含む
- ダイアグラム（テキスト形式）を含むと加点

### セクションガイドライン（一例）
1. **導入** — マルチエージェント設計が必要な背景
2. **従来のマルチエージェントパターン**
   - Pipeline パターン（直列接続）
   - Collaborative パターン（並列協調）
   - Tree パターン（階層構造）
3. **GAN スタイルの特徴**
   - Planner エージェント（仕様策定）
   - Generator エージェント（案の生成）
   - Evaluator エージェント（品質評価）
   - ループ（スコア基準を満たすまで繰り返し）
4. **なぜ GAN スタイルが有効か**
   - 品質と効率のバランス
   - 自動改善ループ
   - 人間の介入を最小化
5. **実装のポイント** — ファイル構成、エージェント間データ受け渡し、停止条件 など
6. **実装例** — ECC/commands/gan-build.md 等の参照
7. **まとめ** — GAN スタイルの応用可能性

## 制約・注意事項
- **ECC の実装に基づく**: 実際の ECC/commands/gan-build.md や ECC/agents/gan-*.md を参照して、正確に説明
- **パターン比較は公正に**: GAN スタイルの利点だけでなく、デメリット・適用限界も述べる
- **実装可能性**: 記事を読む開発者が自分のプロジェクトに適用できるレベルの具体性

## 評価の観点
- マルチエージェント設計の複数パターンを明確に区別できているか
- GAN スタイルの実装が詳しく説明されているか
- なぜ GAN スタイルが有効か、理論的・実践的に説明できているか

## 完成イメージ
読者が「自分たちのシステムにはどのパターンが適しているか」を判断でき、
必要に応じて GAN スタイルを採用する動機付けができるような記事。
