## 実装時の工夫

### エージェント設計の現実性確保
- セキュリティ脆弱性検出：硬化情報、SQL インジェクションなど既知パターン中心
- 命名規則チェック：camelCase / PascalCase / snake_case の確認
- パフォーマンス検査：実行効率、メモリ効率の定義
- 設計原則：SOLID 原則への準拠
- テスト：カバレッジ率と質の両立

### エージェント間通信の具体化
- JSON スキーマで統一的な入出力フォーマット
- `upstream_findings` による先行エージェント結果の引き継ぎ
- `max_context_size_mb` による context 上限設定

### 実装時の落とし穴への対応
- エージェント間コンテキスト喪失 → スキーマ統一、事前テスト
- 実行時間爆発 → パラレル実行可能な観点の分離
- 誤検知増加 → ルール緩和、エージェント責務削減
