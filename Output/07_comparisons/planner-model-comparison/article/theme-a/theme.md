# テーマA: 記事執筆

## タイトル
「Claude Code のサブエージェントを使ってコードレビューを自動化する方法」

## 背景・目的
Claude Code は複数のサブエージェントを連携させることで、複雑なタスクを自動化できます。
特にコードレビューは反復的で、観点が決まった作業なので、マルチエージェント化に向いています。
このタスクでは、「どのようにしてコードレビューエージェント群を設計し、運用するか」を実践的に説明する技術記事を執筆してください。

## 想定読者
- Claude Code を使い始めたソフトウェアエンジニア
- AI を開発ツールとして活用したい開発者
- マルチエージェント設計に興味がある人

## 要求事項

### 必須要素
- **文字数**: 2000～3000文字（日本語）
- **構成**: 導入 → 問題背景 → ソリューション → 実装方法 → まとめ
- **コード例**: 実際に使える設定ファイル・エージェント定義の例を最低2つ以上含める
- **実践性**: 読者がこの記事を読んで、実際に自分のプロジェクトに適用できるレベルの具体性

### 出力形式
- Markdown 形式
- テーブル・リスト・コードブロックを含む
- 見出しは H1 から始める（最上位は H1）

### セクションガイドライン（一例）
1. **導入** — なぜコードレビューを自動化したいのか
2. **既存のレビュー方法の課題** — 手動レビューの限界
3. **マルチエージェント設計** — どんなエージェントを作るか
4. **実装のポイント** — spec.md の書き方、エージェント間連携、評価基準など
5. **実装例** — コード / 設定ファイル
6. **運用のコツ** — 実際に使う際の注意点
7. **まとめ** — このアプローチの可能性と今後

## 制約・注意事項
- **架空の機能は書かない**: Claude Code 実際の仕様に基づいた内容にする
- **実装が必要なわけではない**: 記事は執筆するが、実際にシステムを構築する必要はない
- **信頼性**: 根拠のない主張をしない。確信がない場合は「おそらく」「検討の余地がある」と明記する
- **プライバシー**: 実際のプロジェクトやコードレビューの例を含める場合は、個人・企業情報を隠す

## 評価の観点
- Must-Have 要件をすべて満たしているか（文字数、コード例、実践性）
- 読者にとって有用で実行可能な内容か
- 論理構成が明確か、説得力があるか

## 完成イメージ
読み終えた読者が「自分たちのチームでもマルチエージェントコードレビューを実装してみたい」と思えるような、具体的で実践的な記事。
