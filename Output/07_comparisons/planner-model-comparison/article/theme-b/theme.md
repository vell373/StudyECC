# テーマB: 記事執筆

## タイトル
「コンテキストエンジニアリング入門：プロンプトより強力な設計技術」

## 背景・目的
AI エンジニアリングにおいて「良いプロンプト」は成功の重要な要素ですが、プロンプトよりも重要な（あるいは同等以上に重要な）概念として「コンテキスト設計」があります。
ECC（EverythingClaudeCode）や Claude Code のフック・ルール・スキル設計などは、すべてコンテキストエンジニアリングの実践例です。
このタスクでは、「コンテキストエンジニアリングとは何か」「なぜプロンプトだけで足りないのか」「どう実践するか」を説明する入門記事を執筆してください。

## 想定読者
- Claude API や Claude Code を使い始めた開発者
- プロンプトエンジニアリングの限界を感じている人
- AI システムの設計・運用をより体系的にしたい人

## 要求事項

### 必須要素
- **文字数**: 2000～3000文字（日本語）
- **構成**: 導入 → プロンプトエンジニアリングの限界 → コンテキストエンジニアリングの定義 → 実践例 → まとめ
- **具体例**: ECC / Claude Code の実装例（ルール、スキル、フック、エージェント設定など）を最低2つ以上引用
- **比較視点**: プロンプトとコンテキストをどう使い分けるか、具体的に

### 出力形式
- Markdown 形式
- 比較表を含む（プロンプト vs コンテキスト）
- 実装例はコードブロックで表示

### セクションガイドライン（一例）
1. **導入** — 「良いプロンプト」だけでは足りない理由
2. **プロンプトエンジニアリングの限界** — 単一セッションの中での最適化に留まる、など
3. **コンテキストエンジニアリングとは** — 定義・範囲・重要性
4. **ECC / Claude Code での実例**
   - ルール（rules/）の役割
   - スキル（skills/）の設計
   - フック（hooks/）による自動化
   - エージェント間のコンテキスト伝播
5. **実装のポイント** — コンテキスト設計をどう進めるか
6. **まとめ** — プロンプト + コンテキストの組み合わせの力

## 制約・注意事項
- **ECC の実際の実装に基づく**: 架空の機能ではなく、実際に ECC に存在する実装を参照する
- **初心者向け**: 「コンテキストエンジニアリング」という言葉が新しい人にも理解できるように丁寧に説明
- **説得力**: なぜコンテキスト設計が重要か、理論的に説明する

## 評価の観点
- 「コンテキストエンジニアリング」という概念を明確に説明できているか
- ECC / Claude Code の実例が説得力を持っているか
- 初心者でも読み進められる難易度か

## 完成イメージ
読者が「プロンプトだけに頼るのではなく、システム全体でコンテキストを設計する」という発想に目覚める記事。
