# ポモドーロタイマー + タスク管理

## 概要

**ポモドーロ・テクニック（25分集中 + 5分休憩）とシンプルなタスク管理を統合したウェブアプリケーション**

このアプリケーションは、時間管理とタスク優先度を同時に処理し、開発者やナレッジワーカーが効率的に仕事を進められるよう支援します。

**特徴:**
- 🍅 ポモドーロ・テクニック実装（25分/5分の自動切り替え）
- 📋 シンプルなタスク管理（追加・完了・削除・優先度設定）
- 💾 LocalStorage による自動保存（オフライン完全対応）
- 🌙 ダークモード対応
- 📊 進捗表示（プログレスバー、セッションカウンター）
- 📱 レスポンシブデザイン（PC・タブレット・モバイル対応）
- ♿ アクセシビリティ対応（キーボードナビゲーション、スクリーンリーダー対応）

## 使い方

### 起動方法

1. `index.html` をブラウザで開く
   ```
   open index.html
   ```
   または、対応するウェブサーバーで提供

2. または、ブラウザの開発者ツール（DevTools）の Console でテスト可能

### 基本操作

#### タイマー操作

| 操作 | 説明 |
|------|------|
| **開始ボタン** | 25分のポモドーロセッションを開始（スペースキーでも可） |
| **一時停止ボタン** | タイマーを一時停止（スペースキーでも可） |
| **リセットボタン** | タイマーを初期状態（25:00）に戻す |
| **スペースキー** | 開始/一時停止の切り替え |

#### タスク管理

| 操作 | 説明 |
|------|------|
| **テキスト入力欄** | タスク名を入力 |
| **優先度ボタン** | 高（赤）/ 中（黄）/ 低（緑）から選択 |
| **追加ボタン** | タスクを追加（Enterキーでも可） |
| **チェックボックス** | タスクを完了 |
| **✕ボタン** | タスクを削除 |

#### その他

| 操作 | 説明 |
|------|------|
| **🌙/☀️ ボタン** | テーマをライト/ダークに切り替え |
| **本日をリセット** | 本日のセッションカウントをリセット |

## エッジケース対応

| ケース | 対応 |
|-------|------|
| **長時間離席** | 日付変更時に自動リセット |
| **LocalStorage 満杯** | try-catch でキャッチ、ユーザーに警告 |
| **ブラウザタブ非アクティブ** | setInterval で動作継続（精度に注意） |
| **複数タブでの同時実行** | 各タブが独立して動作（同期なし） |
| **システム時刻の巻き戻し** | 相対計算のため影響なし |
| **モバイルでの画面回転** | Flexbox で自動調整 |
| **Safari のバックグラウンド制限** | タイマー一時停止時に「一時停止中」と表示 |

## トラブルシューティング

### タイマーが動作しない

1. ブラウザのコンソール（F12）で JavaScript エラーを確認
2. LocalStorage が有効か確認（DevTools → Application → Local Storage）
3. ブラウザを再起動

### タスクが保存されない

1. LocalStorage の容量を確認（DevTools → Application → Local Storage）
2. 古いタスクを削除
3. `本日をリセット` ボタンで日付をリセット

### 音声通知が聞こえない

1. ブラウザのボリュームを確認
2. ブラウザの通知設定を確認（Chrome → 設定 → サイト設定 → 通知）
3. Web Audio API が有効か確認

### ダークモードが反映されない

1. ブラウザキャッシュをクリア（Ctrl+Shift+Delete）
2. LocalStorage をクリア（DevTools → Application → Clear Storage）
3. ページをリロード（Ctrl+F5）

## 開発メモ

### コード品質

- **関数分割**: 全関数が 50行以下（複雑な処理は分割済み）
- **命名規則**: camelCase（関数・変数）、kebab-case（CSS）
- **エラーハンドリング**: try-catch で LocalStorage エラーを捕捉
- **DRY 原則**: 重複コードなし、共通処理は関数化

### 設計判断

1. **App クロージャ**: グローバル汚染を避け、プライベート状態を管理
2. **render() 関数**: UI 全体を再描画（パフォーマンス許容内）
3. **setInterval ベース**: requestAnimationFrame よりシンプル、秒単位の精度で十分
4. **CSS 変数**: テーマ切り替えを簡潔に実装
5. **LocalStorage キー1個**: スキーマ変更時の管理が容易

### 既知の制限・今後の改善案

1. **複数タブでの非同期化**: storage イベントリスナーで同期可能（Phase 2）
2. **タスク編集機能**: インライン編集UI を追加（Nice-to-Have）
3. **ネットワーク同期**: サーバー連携（クラウド保存）
4. **Service Worker**: オフライン時の Service Worker 対応（オプション）
5. **グラフ表示**: Canvas または Chart.js での週間統計（Nice-to-Have）

## 参考

### ポモドーロ・テクニック

ポモドーロ・テクニックは、Francesco Cirillo が開発した時間管理術です。

```
1. 25分の集中セッション（ポモドーロ）
2. 5分の休憩
3. 3回繰り返し後、15分の長休憩
```

このアプリケーションは、このテクニックを実装し、タスク管理と組み合わせて使います。

### 推奨使用方法

```
朝:
  → タスクリストに本日のタスクを追加
  → ポモドーロセッション開始

セッション中:
  → 現在のタスクに集中
  → 25分完了後、自動で休憩へ

1日:
  → 4セッション = 約2時間
  → 3〜4セッションで半日の作業を完結
```
