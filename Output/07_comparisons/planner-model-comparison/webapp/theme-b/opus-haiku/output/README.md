# ポモドーロタイマー + タスク管理ウェブアプリ

ポモドーロ・テクニック（25分集中 + 5分休憩）を実践しながら、シンプルなタスクリスト管理を同時に行えるウェブアプリです。

ブラウザで `index.html` をダブルクリックするだけで動作します。外部ライブラリやサーバは不要です。

## 起動方法

1. このフォルダ内の `index.html` をダブルクリック、またはブラウザで開く
2. ブラウザタブで表示されたら準備完了

## 機能一覧

### Must-Have（実装済み）

#### 1. ポモドーロタイマー本体
- **25分集中タイム** → 自動で **5分休憩** へ切り替わる
- **5分休憩終了後** → **25分集中タイム** へ戻る
- **開始 / 一時停止 / リセット** ボタンで操作
- **正確な時間計測**: タブが非アクティブ時も誤差 ±2秒以内
- **フェーズ終了時に通知**:
  - Notification API（デスクトップ通知）
  - 音声アラート（ブラウザ的に許可されている場合）
  - Alert ダイアログ（フォールバック）

#### 2. セッションカウンター
- 25分セッション完了ごとに **「総セッション数」が +1**
- **「本日のセッション数」** を別途表示（日付変更時自動リセット）
- 4セッション完了後に **「15分の長休憩」を提案** → ユーザ選択で開始

#### 3. タスクリスト CRUD
- **追加**: テキスト入力 + ボタンクリック、または Enter キー
- **完了**: チェックボックス操作で切り替え → 取り消し線が表示される
- **削除**: 削除ボタンで即座に削除
- **優先度設定** (Should-Have): 高（赤）/ 中（黄）/ 低（青）で色分け

#### 4. LocalStorage による永続化
- **保存対象**:
  - タスク一覧（テキスト、完了状態、優先度）
  - セッション累計数
  - 本日のセッション数（日付付き）
  - ダークモード設定
- **動作**:
  - ブラウザリロード後に状態が完全復元される
  - ページを閉じて再度開いても復元される
  - LocalStorage が無効でも UI上は動作継続（警告表示）

#### 5. 3ファイル構成での動作
- `index.html` （HTML構造）
- `style.css` （スタイリング）
- `script.js` （ロジック）
- `README.md` （このファイル）
- ビルド・サーバ不要

### Should-Have（実装済み）

#### 1. タスク優先度設定
- 追加時に **「高 / 中 / 低」** をセレクトボックスで指定
- リスト上で **左ボーダーの色** で優先度を視覚化

#### 2. 本日の記録パネル
- タイマーセクション右側に表示:
  - 「総セッション数」
  - 「今日のセッション数」
- 日付が変わると自動リセット

#### 3. ダークモード切り替え
- ヘッダー右上の 🌙 ボタンで ON/OFF
- CSS 変数で全体のテーマを切り替え（背景・テキスト・アクセント色）
- 設定は LocalStorage に保存

#### 4. 進捗プログレスバー
- SVG 円形グラフで現在のセッション残時間を視覚化
- 1秒ごとに滑らかに更新

## 操作方法

### タイマー操作

| ボタン | 動作 |
|--------|------|
| **開始** | 25分タイマーを開始。自動で「集中タイム」→「短い休憩」を繰り返す |

| 操作 | 動作 |
|------|------|

| **Enter キー** | テキスト入力後に Enter で即追加 |
| **チェックボックス** | タスクの完了・未完了を切り替え |
| **削除ボタン** | タスクを削除 |
| **優先度セレクト** | 追加前に優先度（高/中/低）を選択 |

### テーマ切り替え

- ヘッダー右上の 🌙 ボタンで **ダークモード** ON/OFF

## 設計上の特徴

### タイマーの精度確保

タブが非アクティブ時にブラウザの `setInterval` が遅延する問題に対応:

```javascript
// 開始時刻を記録
state.timer.startTime = Date.now();

// 更新時に実際の経過時間を計算
const elapsed = Math.floor((Date.now() - state.timer.startTime) / 1000);
const remaining = state.timer.totalSeconds - elapsed;
```

### 状態管理（Flux 風）

- グローバル `state` オブジェクトが唯一の真実のソース
- 変更後に必ず `render()` と `persist()` を呼ぶ
- 予測可能で保守しやすい

### コード構成

| 関数 | 責務 |
|------|------|
| `startTimer()` | タイマー開始 |
| `pauseTimer()` | タイマー一時停止 |
| `resetTimer()` | タイマーリセット |
| `updateTimer()` | 時間更新（毎フレーム）|
| `completePhase()` | フェーズ終了処理 |
| `addTask()` / `deleteTask()` / `toggleTask()` | タスク CRUD |
| `render()` | UI 全更新 |
| `persist()` | LocalStorage 保存 |
| `loadState()` | LocalStorage 復元 |

各関数は単一責任で、50行以内に収まっています。

## エッジケース対応

### タブが閉じていた間にフェーズが終了した場合

起動時に以下のチェックを実施:

```javascript
if (state.timer.isRunning && state.timer.startTime) {
    const elapsed = Math.floor((Date.now() - state.timer.startTime) / 1000);
    if (elapsed >= state.timer.totalSeconds) {
        completePhase(); // 終了済みとして処理
    }
}
```

### LocalStorage が満杯またはプライベートモード

`try/catch` で囲み、エラー時は画面に警告を表示。メモリ上の状態で動作継続。

### 空のタスク名

追加時に空白のみのチェックを実施。エラーメッセージを表示してリターン。

### 入力フォーカスとキーボード操作

- Tab キー でボタン・入力欄間を移動可能
- Enter キー でタスク追加
- すべての操作がキーボードのみで完結

## トラブルシューティング

### タイマーが動かない

1. ブラウザコンソール（F12）でエラーがないか確認
2. JavaScriptが有効か確認
3. ブラウザをリロード（Ctrl+R）

### タスクが保存されていない

1. LocalStorage が無効でないか確認（プライベートモード等）
2. ブラウザのストレージ制限を超えていないか確認
3. コンソールで警告メッセージを確認

### 通知音が出ない

- ブラウザの自動再生ポリシーで Audio の再生が制限されている可能性
- 初回操作（開始ボタン）後の通知では音が出る設定が必要
- または Notification API で デスクトップ通知に切り替わります

### ダークモードが反映されない

1. ブラウザをリロード
2. 設定が LocalStorage に保存されているか確認（DevTools > Application > LocalStorage）
