# 設計仕様書: ポモドーロタイマー + タスク管理ウェブアプリ

## 概要
ポモドーロ・テクニック（25分集中 + 5分休憩のサイクル）を実践しながら、シンプルなタスクリスト管理を同時に行えるウェブアプリを作成する。HTML/CSS/JavaScript のみで構成し、外部ライブラリやサーバを使わず、LocalStorage によりタスクとセッション履歴をブラウザに保存する。完成形は「ブラウザでindex.htmlを開くだけで動作する、毎日使いたくなる集中支援ツール」である。タイマーが正確に動作し、タスクの追加・完了・削除・永続化が確実に行えることが最重要。Should-Have として優先度設定、ダークモード、進捗のビジュアル表示まで取り込み、洗練されたUIに仕上げる。

## 要件一覧

### Must-Have（実装必須）
1. **ポモドーロタイマー本体**: 
   - 25分集中・5分休憩を自動で切り替える
   - 開始 / 一時停止 / リセット の3ボタンが動作する
   - `mm:ss` 形式でカウントダウン表示される
   - フェーズ終了時に通知音 または `Notification API` または `alert()` フォールバックで通知が出る
   - チェック方法: ストップウォッチで25分=1500秒、5分=300秒であることを確認、タブが非アクティブでも誤差±2秒以内で動作

2. **セッションカウンター**: 
   - 25分セッション完了ごとに +1 し、現在の累計を画面に表示
   - 4セッション完了したら次の休憩を「長休憩(15分)」に切り替える（自動 or ユーザに提案）
   - チェック方法: 4回 25分セッションを早送り（テスト用関数で時間短縮）して長休憩に入ることを確認

3. **タスクリスト CRUD（追加・完了・削除）**:
   - 入力欄 + 追加ボタン または Enter キーで新規タスク追加
   - チェックボックス操作で「完了」状態をトグル → 完了タスクは取り消し線（`text-decoration: line-through`）表示
   - 各タスク行に削除ボタンを設置、押下で即削除（簡易確認は任意）
   - チェック方法: 5件追加 → 2件完了 → 1件削除 で正しい状態に遷移

4. **LocalStorage による永続化**:
   - 保存対象: タスク配列、累計セッション数、本日セッション数（日付付き）、テーマ設定
   - スキーマ例: `{ tasks: [...], sessions: { total: n, today: { date: 'YYYY-MM-DD', count: n } }, settings: {...} }`
   - チェック方法: タスク作成 → ブラウザリロード / タブを閉じて再オープン → 状態が復元される

5. **3ファイル構成での動作**:
   - `index.html`, `style.css`, `script.js`, `README.md` を提供
   - ブラウザで `index.html` をダブルクリックするだけで動作する（ビルド不要）
   - チェック方法: Chrome / Firefox / Safari の最新版で開いて全機能が動く

### Should-Have（あると良い）
1. **タスク優先度設定**: 高（赤）/ 中（黄）/ 低（青）の3段階。追加時にセレクトボックスで指定し、リスト上で色アイコン or 左ボーダーで識別
2. **本日の記録パネル**: 「今日のセッション数 / 今日完了したタスク数」を表示。日付が変わったらリセット（LocalStorage の保存日付と比較）
3. **ダークモード切り替え**: ヘッダー右上にトグルボタン。CSS変数 `--bg`, `--text`, `--accent` 等で配色を切替し、設定を LocalStorage に保存
4. **進捗プログレスバー**: 現在のセッション残時間を視覚化（円形 SVG または横バー、CSS のみで実装可）

### Nice-to-Have（実装できれば）
1. **タスク名のインライン編集**: ダブルクリックで編集モードに切り替え、Enter で保存・Esc でキャンセル
2. **設定: 集中/休憩時間のカスタマイズ**: 25分・5分・15分をユーザが変更可能（テスト容易性も向上）
3. **週次の完了セッション簡易グラフ**: 過去7日間を CSS だけで棒グラフ表示

## 技術スタック（推奨）
- **HTML5（セマンティックタグ）**: `<header>`, `<main>`, `<section>`, `<ul>` を使い構造を明確化。アクセシビリティと可読性のため
- **CSS3（CSS変数 + Flexbox/Grid）**: テーマ切り替えを CSS 変数で簡潔に実装でき、追加ライブラリ不要
- **Vanilla JavaScript (ES2020+)**: 制約上ライブラリ禁止。クラスベースで `Timer`, `TaskStore`, `UIController` に責務分割
- **Web APIs**: `setInterval` / `Date.now()` 併用で時間計測の精度確保、`Notification API`、`Audio` でフェーズ終了通知、`localStorage` で永続化
- **モジュール構成**: `<script type="module">` を使い `script.js` から個別ファイルを import するのは可。簡素にしたい場合は単一ファイルで OK
- **時間計測精度**: `setInterval(fn, 1000)` のみだとタブ非アクティブ時にズレるため、開始時刻を記録して経過時間を `Date.now() - startedAt` で算出する方式を採用

## 実装フェーズ計画

### Phase 1: Must-Have コア機能 — 動くポモドーロタイマー
**目標**: 集中→休憩→セッションカウント→タスク管理→保存 の一連が単一画面で動作する状態
**対象要件**: Must-Have 1〜5
**完了条件**: 
- 25分カウントダウンが正確に動作し、終了で通知 → 5分休憩へ自動遷移
- 4セッション後に長休憩(15分)に切り替わる
- タスクの追加・完了・削除が動作
- リロード後にタスク一覧とセッション数が復元される
- index.html を開くだけで全機能が動く

### Phase 2: Should-Have と仕上げ — 実用性とUX強化
**目標**: 「毎日使いたくなる」UI/UX に仕上げる
**対象要件**: Should-Have 1〜4、可能なら Nice-to-Have 1
**完了条件**:
- 優先度別にタスクが視覚的に区別される
- 本日のセッション数 / 完了タスク数が表示され、日付変わりでリセットされる
- ダークモードがトグルでき、再訪時も維持される
- プログレスバーが滑らかに更新される（少なくとも1秒ごと）
- README.md に起動方法と操作説明が記載されている

## 品質基準（これが揃えば「完成」とみなす）
- タイマーが ±2秒以内の精度で 25分 / 5分 / 15分 を計測する（タブ非アクティブ時も含む）
- ブラウザリロード後にタスク・累計セッション・テーマが完全復元される
- 全ボタン / 入力に対するクリック・Enter操作が直感的に動作する
- コンソールエラー・警告ゼロで起動する
- `script.js` が論理単位で関数分割され、1関数 50行以内、1ファイル 300行以内（必要なら分割）
- README.md に「起動方法」「操作方法」「機能一覧」「既知の制限」が記載されている
- Chrome / Firefox / Safari の最新版で外観崩れなく動作

## 設計上の重要な判断点
- **タイマーの精度確保方法**: `setInterval(1000ms)` 単独ではタブ非アクティブで遅延するため、「開始時刻 + 設定時間」を保持し、`Date.now()` との差分で残時間を算出する方式を採用。表示更新だけが `setInterval` の役割
- **状態管理の構成**: 単一の `state` オブジェクト（`{ timer, tasks, sessions, settings }`）を SSOT とし、変更後に `render()` と `persist()` を呼ぶ単純な「Flux 風」パターン。ライブラリ不要で見通しが良い
- **永続化のタイミング**: 各 mutation 直後に LocalStorage へ書き込む。容量が小さいためデバウンス不要
- **長休憩の扱い**: 4セッション完了後、「長休憩を始めますか？」と確認 → ユーザが選択してから開始する方式（自動より制御感が高い）。alert ではなくモーダル/ボタン UI 推奨
- **通知方法の優先順位**: ① `Notification API`（許可済み）→ ② Audio 再生 → ③ `alert()`（最終フォールバック）。許可状態を起動時に確認し、ユーザに案内
- **テーマファイル分割**: 単一HTMLでも動作可だが、`script.js` `style.css` を分けることで 1ファイル300行制限と保守性を満たす

## 考慮すべきエッジケース
- **タブを閉じている間にフェーズ終了時刻を過ぎた場合**: 起動時に `Date.now()` と `startedAt + duration` を比較し、過ぎていれば「フェーズ完了」として処理してから現在状態を復元
- **LocalStorage が満杯/無効（プライベートモード等）**: try/catch で囲み、失敗時は警告メッセージを画面に表示してメモリ上のみで動作継続
- **同時に複数タブを開いた場合**: `storage` イベントを購読して他タブの変更を取り込む（Phase 2 範囲）。最低限、最後に書き込んだタブの状態が残れば許容
- **入力が空 / 空白のみのタスク**: 追加を拒否し入力欄にフォーカスを戻す
- **極端に長いタスク文字列**: CSS で `text-overflow: ellipsis` または最大文字数制限（例: 200文字）
- **日付変更を跨いだ稼働**: 「本日のセッション数」は保存された日付と現在日付を比較し、異なれば 0 にリセット
- **通知許可が拒否されている**: 起動時に検知し、Audio 通知に自動切替。UI に「通知が無効です」とインジケータ表示
- **音声がブラウザポリシーで再生不可（自動再生制限）**: 初回ユーザ操作（開始ボタン押下）時に Audio をアンロックする処理を入れる
- **リセットボタン押下時の挙動**: 現在のフェーズの残時間のみリセットし、累計セッション数は保持（誤操作対策）。「全リセット」は別ボタンで提供してもよい

## 制約・注意事項
- 外部ライブラリ・CDN 依存を一切使用しないこと（CSS リセット程度の自作は OK）
- ビルドツール不要・サーバ不要で `index.html` ダブルクリックで動くこと
- ECC フォルダは変更対象外（このタスクのアウトプットは指定パス配下のみ）
- コード規約: 変数/関数 camelCase、クラス PascalCase、定数 UPPER_SNAKE_CASE、ファイル名 kebab-case
- セキュリティ: タスク名を innerHTML で挿入せず textContent を使用（XSS 防止）
- コメントは「なぜ」を書く。「何を」だけのコメントは不要
