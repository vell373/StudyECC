# シンプルな習慣トラッカー — 設計仕様書（opus-haiku パターン）

## 概要

このプロジェクトは、毎日の習慣を追跡し連続実施日数を可視化するシンプルなウェブアプリケーションです。ユーザーは最大10個の習慣を管理でき、各習慣について「今日実施」ボタンを押下することで本日の実施記録がブラウザのローカルストレージに保存されます。記録がない日が挟まると連続日数がリセットされ、モチベーション維持をサポートする視覚的フィードバック（連続日数・最高連続日数の表示）を提供します。外部ライブラリを一切使用しない（HTMLとCSSのみ、JavaScriptは標準API）シンプルで保守性の高い実装を目指します。

## 要件一覧

### Must-Have（実装必須）

| 要件 | 説明 | 受け入れ条件 |
|------|------|-----------|
| 1. 習慣の追加 | テキスト入力で習慣名を入力し「追加」ボタンで習慣リストに登録 | 最大10個まで管理可能 / 習慣名がテキストフィールドに反映 / 追加ボタン押下でリストに加算 |
| 2. 日々のチェックイン | 習慣ごとに「今日実施」ボタンで本日の実施記録を保存 | 1日1回のカウント / 重複押下で増加しない / ボタン状態が「実施済み」に変化 |
| 3. 連続日数の表示 | 各習慣の現在の連続日数と過去最高連続日数を表示 | 記録がない日で連続日数が0にリセット / 大きく見やすく表示 / 最高記録も記録・保持 |
| 4. ローカルストレージ保存 | 習慣と実施記録をブラウザに永続化 | ページリロード後も記録が残存 / ブラウザ再起動後も復元 |

### Should-Have（あると良い）

| 要件 | 説明 | 優先度 |
|------|------|--------|
| 1. 習慣削除 | 不要な習慣を削除できる | 高 |
| 2. カラーコード | 習慣ごとに色を選択・表示（視覚的識別） | 中 |
| 3. 週ビュー | 過去7日間の実施状況をカレンダー表示 | 中 |
| 4. 統計サマリー | 本週の実施率を表示（「完璧な週！」など） | 中 |

### Nice-to-Have（実装できれば）

| 要件 | 説明 |
|------|------|
| 1. 削除確認ダイアログ | 習慣削除時に確認ダイアログで誤削除防止 |
| 2. 習慣名編集 | 習慣名を後から変更可能 |
| 3. スナップショット機能 | 月単位の記録を表示・ダウンロード |
| 4. ダークモード | ライト/ダーク切り替え（OS設定に連動） |

## エッジケース

### 日付・時間関連

1. **タイムゾーン対応**
   - ユーザーの場所によって「今日」の定義が異なる可能性
   - **対応**: `new Date()` で取得した日付を ISO 8601 形式 (YYYY-MM-DD) で保存
   - LocalStorage キーに日付文字列を使用し、時間部分を完全に無視

2. **サマータイム / DST**
   - ブラウザの時刻設定が変わった場合の連続日数計算
   - **対応**: Date オブジェクトから getFullYear / getMonth / getDate で日付のみ抽出、時刻は参照しない

3. **複数ブラウザ / 複数デバイス**
   - 別のブラウザで同じサイトを開いた場合、別々のストレージ
   - **対応**: LocalStorage はドメイン単位なため、別ブラウザとは同期されない（設計の制約として明記）

### ストレージ関連

4. **LocalStorage 満杯**
   - ブラウザのストレージクォータ (5～10MB) に達した場合
   - **対応**: try-catch で JSON.stringify 時のエラーをキャッチ、ユーザーに警告表示

5. **LocalStorage クリア / ブラウザキャッシュ削除**
   - ユーザーが誤ってキャッシュ削除した場合、全データが消失
   - **対応**: データ消失は避けられないが、初回起動時ガイドを表示

6. **破損データの復元**
   - LocalStorage に不正な JSON が格納されている場合
   - **対応**: JSON.parse() エラーをキャッチ、デフォルト値でリセット、コンソール警告を出力

### ビジネスロジック関連

7. **習慣の重複追加**
   - 同じ名前の習慣を複数回追加した場合
   - **対応**: 現在の実装では許容（設計仕様で「重複チェックなし」が正）
   - オプション: Should-Have で重複検知機能を追加可能

8. **連続日数のカウント開始タイミング**
   - 習慣を追加した初日をカウント対象にするか?
   - **対応**: 初日に「今日実施」ボタンを押した日から連続日数 = 1 として開始

9. **午後23:59に「今日実施」を押した直後に日付が変わった場合**
   - ローカルタイムの日付と実装の日付判定のズレ
   - **対応**: 取得した日付が「今日」として保存されるため、ズレはブラウザの時刻設定に依存

10. **習慣削除後の統計データ**
    - 削除した習慣の過去の実施記録は失われる
    - **対応**: 削除は「論理削除（フラグ）」ではなく「完全削除」として実装、リカバリ機能なし（Undo 機能は Nice-to-Have）

### ユーザー操作関連

11. **空の習慣名追加**
    - テキストフィールドが空のまま「追加」ボタンを押した場合
    - **対応**: 入力検証（trim 後の長さチェック）で空文字列を拒否、エラーメッセージ表示

12. **大量の習慣データ保存**
    - 習慣数は最大10個に制限されているが、実施記録は無制限に増加
    - **対応**: 年単位で古いデータが蓄積する可能性があるため、長期的には月単位でアーカイブ機能の検討を推奨（現フェーズでは対象外）

13. **複数ウィンドウ / タブでの同時編集**
    - 複数タブでアプリを開き、同時に「今日実施」を押した場合
    - **対応**: LocalStorage は tab 間で同期されるため、後から開いたタブが新しいデータを上書き（race condition の可能性あり）
    - **改善案**: Version フィールドを追加し、競合検知機能を実装（Nice-to-Have）

### 表示・レイアウト関連

14. **超長い習慣名**
    - 習慣名が50文字を超える場合、レイアウト崩れの可能性
    - **対応**: CSS で max-width / word-break: break-word を設定、省略表示オプション追加

15. **小さい画面（モバイル）での表示**
    - 週ビューが横スクロール必要な場合
    - **対応**: モバイル時は週ビューを縦方向スクロール / スワイプ対応、or 日数を動的に削減

## 実装のヒント・注意点

### JavaScript 日付処理

```javascript
// ✅ 推奨: 時刻を無視した日付比較
function getTodayString() {
  const now = new Date();
  return now.getFullYear() + '-' + 
         String(now.getMonth() + 1).padStart(2, '0') + '-' + 
         String(now.getDate()).padStart(2, '0');
}

// ❌ 非推奨: タイムスタンプで保存（タイムゾーン問題発生）
const timestamp = new Date().getTime();
```

### LocalStorage 管理

```javascript
// ✅ 推奨: キー名を定数化
const STORAGE_KEY = {
  HABITS: 'habits_data',
  RECORDS: 'habits_records'
};

// LocalStorage 読み込み時
try {
  const habits = JSON.parse(localStorage.getItem(STORAGE_KEY.HABITS)) || [];
} catch (e) {
  console.error('LocalStorage corruption:', e);
  // デフォルト値で復帰
}
```

### 状態管理

```javascript
// ✅ 推奨: アプリケーション全体の状態を1つのオブジェクトで管理
const appState = {
  habits: [], // 習慣リスト
  records: {} // 習慣ID -> [日付リスト]
};

// UI 更新は appState の変更後に呼び出す
function addHabit(name, color) {
  appState.habits.push({ id, name, color });
  saveToStorage();
  render();
}
```
