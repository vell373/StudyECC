# シンプルな習慣トラッカー

毎日の習慣を追跡し、連続実施日数を可視化するシンプルなウェブアプリケーションです。外部ライブラリを一切使用しない、純粋な HTML/CSS/JavaScript で実装されています。

## 機能一覧

### Must-Have（必須機能）

1. **習慣の追加**
   - テキスト入力で習慣名を入力し、「追加」ボタンで習慣リストに登録
   - カラーコード（色選択）を習慣ごとに設定可能
   - 最大10個の習慣を管理可能
   - 入力検証（空文字列は拒否）

2. **日々のチェックイン**
   - 習慣ごとに「今日実施」ボタンで本日の実施記録を保存
   - 1日1回のカウント（重複押下で増加しない）
   - ボタン状態が「実施済み」に変化して視覚的フィードバック

3. **連続日数の表示**
   - 各習慣の現在の連続日数と過去最高連続日数を大きく表示
   - 記録がない日で連続日数が 0 にリセット
   - 最高記録も記録・保持

4. **ローカルストレージ保存**
   - 習慣と実施記録をブラウザに永続化
   - ページリロード後も記録が残存
   - ブラウザ再起動後も復元

### Should-Have（拡張機能）

1. **習慣削除**
   - 習慣ごとに「削除」ボタンで不要な習慣を削除
   - 削除確認ダイアログで誤削除防止

2. **カラーコード**
   - 習慣追加時に色選択UI で色を設定
   - 選択色を習慣ごとに保存・表示
   - リスト表示時に左ボーダー色として反映

3. **週ビュー**
   - 過去7日間の実施状況をグリッド表示
   - 日付ヘッダ（月日 / 曜日）付き
   - 習慣ごとに実施状況を「✔」「○」「-」で表示

4. **統計サマリー**
   - 本週（過去7日）の実施率を計算・表示
   - 「完璧な週！」「好調な週 (80%以上)」「頑張ろう」などメッセージ表示
   - 実施数 / 総数も表示

## セットアップ

### 必要な環境

- モダンウェブブラウザ（Chrome 90+、Firefox 88+、Safari 14+、Edge 90+）
- ローカルサーバー（任意）

### インストール

1. `output/` ディレクトリ内のすべてのファイルをダウンロード
2. ブラウザで `index.html` を開く

#### 簡易ローカルサーバーで実行する場合

**Python 3:**
```bash
cd output/
python -m http.server 8000
```

ブラウザで http://localhost:8000 にアクセス

**Node.js（http-server）:**
```bash
npx http-server output/
```

## 使い方

### 1. 習慣を追加する

1. 「習慣を追加」セクションの入力欄に習慣名を入力
2. 色選択ボタンで習慣の色を選択（オプション）
3. 「追加」ボタンをクリック
4. 習慣がリストに追加されます

**制限:**
- 最大10個まで登録可能
- 習慣名は空でない（1文字以上50文字以内）

### 2. 日々の習慣をチェックインする

1. 習慣リストの「今日実施」ボタンをクリック
2. ボタンが「✓ 実施済み」に変わり、グレーアウト（disabled）
3. 連続日数が更新される

**注意:**
- 1日1回のみカウント（同じ日に複数回押下しても1回のみ）
- 日付は ブラウザのローカルタイムに基づく

### 3. 連続日数を確認する

- **「連続日数」**: 今日からさかのぼった連続実施日数
- **「最高記録」**: これまでの最高連続日数
- 記録がない日が挟まるとリセット（連続日数 = 0）

### 4. 習慣を削除する

1. 習慣の「削除」ボタンをクリック
2. 確認ダイアログで「OK」を選択
3. 習慣がリストから削除される

**注意:**
- この操作は取り消せません
- 削除した習慣のすべての履歴が失われます

### 5. 週ビューで進捗を確認する

- 「週ビュー」セクションに過去7日間の実施状況が表示
- ✔：実施済み、○：未実施、-：未来の日付
- 習慣ごとの行で一週間の進捗を一覧確認

### 6. 統計サマリーを確認する

- 「本週の統計」セクションに実施率を表示
- メッセージ例：
  - 100% → 「完璧な週！」
  - 80-99% → 「好調な週 (80%以上)」
  - 50-79% → 「頑張ろう (50%以上)」
  - 0-49% → 「もう少し頑張れば (50%未満)」

## 技術仕様

### ファイル構成

```
output/
├── index.html      — メインHTML（構造）
├── style.css       — スタイルシート（CSS3 / Flexbox / Grid）
├── script.js       — JavaScriptロジック（ES6以降）

```

### 技術スタック

- **HTML5**: セマンティック構造
- **CSS3**: Flexbox / Grid / CSS カスタムプロパティ
- **JavaScript**: ES6 以降（標準API のみ）
- **ストレージ**: LocalStorage API

### データ構造

#### appState
```javascript
{
  habits: [
    { id: "habit_1234567890", name: "朝ジョギング", color: "#4CAF50" },
    ...
  ],
  records: {
    "habit_1234567890": ["2026-05-09", "2026-05-08", "2026-05-07", ...],
    ...
  }
}
```

#### LocalStorage キー
- `habits_data`: 習慣オブジェクトの配列（JSON）
- `habits_records`: 習慣ID → 実施日付配列のマップ（JSON）

### 日付処理

- **形式**: YYYY-MM-DD（ISO 8601、タイムゾーン非依存）
- **時刻**: 完全に無視（日付のみで比較）
- **実装**: JavaScript Date オブジェクトのみ使用

**例:**
```javascript
getTodayString()  // "2026-05-09"
getDateString(1)  // "2026-05-08"（昨日）
```

### 連続日数計算

連続日数は、**今日から遡る** で計算：
1. 本日の記録があるか確認
2. 昨日の記録があるか確認
3. 記録がない日で計算を止める

**例:**
- 記録: ["2026-05-09", "2026-05-08", "2026-05-07"] → 連続日数 = 3
- 記録: ["2026-05-09", "2026-05-07"] → 連続日数 = 1（05-08 の記録がない）

## エッジケース対応

### タイムゾーン

- アプリケーションはブラウザのローカルタイムを使用
- ユーザーのタイムゾーン設定に自動対応
- 異なるタイムゾーン間での同期は行わない（LocalStorage の制限）

### LocalStorage 満杯

- `localStorage.setItem()` 時にエラーをキャッチ
- ユーザーにアラート表示
- データは保存されない（既存データは保持）

### 破損データ復元

- `JSON.parse()` エラーはキャッチ
- コンソール に警告を出力
- デフォルト値（空の配列）でリセット

### 複数タブ / ウィンドウ

- `storage` イベントで他タブの変更を検知
- 自動的にデータを再読み込み
- race condition の可能性あり（同時更新時）

### 習慣の重複追加

- 現在の実装では、同じ名前の習慣を複数登録可能
- ビジネス要件により重複チェック機能を追加可能

## トラブルシューティング

### データが表示されない

1. ブラウザのコンソール（F12）でエラーを確認
2. LocalStorage が有効か確認：
   ```javascript
   localStorage.getItem('habits_data')  // 開発者ツール > Console で実行
   ```
3. ブラウザキャッシュをクリア（Ctrl+Shift+Delete）

### 習慣の数が更新されない

- ページをリロード（F5）
- LocalStorage が満杯の可能性→古いブラウザキャッシュを削除

### 色が反映されない

- ブラウザが色入力対応か確認
- Safari のバージョン確認（14以上推奨）

### 連続日数がリセットされた

- 記録がない日が挟まっていないか確認
- 日付がシステムで正しく認識されているか確認（タイムゾーン）

## 制限事項

1. **複数ブラウザ / デバイス間での同期なし**
   - 異なるブラウザ・デバイス間ではデータが独立
   - 解決策：クラウド同期機能の追加が必要

2. **オフライン限定**
   - インターネット接続不要
   - サーバーへのバックアップなし

3. **データバックアップなし**
   - ブラウザのキャッシュ削除でデータが失われる
   - 解決策：エクスポート機能の追加が必要
