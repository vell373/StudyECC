# シンプルな習慣トラッカー — Haiku-Opus パターン設計仕様書

## 概要

シンプルな習慣トラッカーは、毎日繰り返す習慣（運動、読書、瞑想など）を追跡し、連続実施日数をカウントするウェブアプリケーションです。ユーザーは習慣を定義し、毎日「実施した」をマークすることで、モチベーションを維持できます。本パターンでは、Haiku（簡潔な設計思想）と Opus（豊かな実装能力）を組み合わせ、最小限のコード数で最大限の価値を提供する実装を目指します。

## 要件一覧

### Must-Have（必須）

| ID | 要件 | 説明 | 受け入れ条件 |
|----|----|------|---------|
| MH-1 | 習慣の追加 | ユーザーが新しい習慣を定義・登録できる | テキスト入力で習慣名を入力し、追加ボタンで最大10個まで登録可能 |
| MH-2 | 日々のチェックイン | 習慣を実施した日をマークできる | 習慣ごとに「今日実施」ボタンがあり、押下で本日の実施記録が保存される |
| MH-3 | 重複チェック | 1日1回のみカウント | 同じ習慣を同じ日に複数回マークしても、1回分のみカウント |
| MH-4 | 連続日数表示 | 各習慣の連続実施日数を表示 | 連続日数が大きく見やすく表示される。記録なし日が挟まると連続日数がリセット |
| MH-5 | 最高記録表示 | 最高連続日数を記録・表示 | 過去の最高連続日数をバッジ等で表示 |
| MH-6 | ローカルストレージ保存 | 習慣と記録をブラウザに保存 | ページリロード・ブラウザ再起動後も記録が残っている |

### Should-Have（あると良い）

| ID | 要件 | 説明 | 優先度 |
|----|----|------|--------|
| SH-1 | 習慣削除 | 不要な習慣を削除できる | 高 |
| SH-2 | カラーコード | 習慣ごとに色を選択できる | 高 |
| SH-3 | 週ビュー | 過去7日間の実施状況をカレンダー表示 | 中 |
| SH-4 | 統計サマリー | 本週の実施率と完成度メッセージを表示 | 中 |

### Nice-to-Have（実装できれば）

| ID | 要件 | 説明 |
|----|----|------|
| NH-1 | 削除確認ダイアログ | 習慣削除時に確認を求める |
| NH-2 | 習慣名編集 | 習慣名を後から変更できる |
| NH-3 | スナップショット機能 | 月単位の記録を表示 |
| NH-4 | ダークモード | ライト/ダーク切り替え |

## 技術スタック（推奨）

### 言語・環境
- **HTML5** — セマンティックマークアップ
- **CSS3** — フレキシブルレイアウト（Grid / Flexbox）
- **JavaScript（ES6+）** — 状態管理、日付計算、イベントハンドリング
- **LocalStorage API** — 永続化ストレージ

### 制約条件
- 外部ライブラリ不可（jQuery、Moment.js、React等一切不可）
- Date オブジェクトのみ使用（日付計算）
- モダンブラウザで動作（Chrome, Firefox, Safari, Edge の最新版）

## エッジケース

本実装で対応すべき例外ケース：

### 日付・タイムゾーン
1. **タイムゾーン境界**: ユーザーのローカル時刻を基準に「今日」を判定する
   - 対応: `new Date().toLocaleDateString()` で日付を正規化
2. **夜中のチェックイン**: 深夜 2 時に「今日」をマークした場合、翌朝の同じアプリでも当日記録として扱う
   - 対応: 記録日を ISO 8601（YYYY-MM-DD）で文字列化し、日付比較時は文字列比較

### ユーザー操作
3. **重複登録**: 同じ習慣名を 2 回追加しようとした場合
   - 対応: バリデーションで既存習慣との重複チェック
4. **空文字入力**: 習慣名が空、またはスペースのみの場合
   - 対応: トリム後に長さチェック、UI に「1文字以上入力してください」メッセージ
5. **最大10個超過**: 11個目を追加しようとした場合
   - 対応: 「最大10個まで」アラートを表示、追加を拒否
6. **連続削除**: 習慣削除後、その記録をどう扱うか
   - 対応: 習慣削除時、その習慣の全記録も同時削除（参照整合性）

### データ・ストレージ
7. **LocalStorage 満杯**: 5MB 超過時の挙動
   - 対応: 古い月の記録を自動削除、またはアラート表示
8. **LocalStorage クリア**: ユーザーが手動で LS をクリアした場合
   - 対応: アプリ起動時に空の初期化、新規ユーザーとして扱う
9. **腐敗データ**: 不正な JSON が保存されている場合
   - 対応: try-catch で parse エラーを捕捉、デフォルト値で復旧

### UI/レイアウト
10. **長い習慣名**: 30字以上の習慣名を入力した場合
    - 対応: 折り返し、または省略表示（ellipsis）
11. **最小ウィンドウ幅**: 256px 未満のウィンドウ
    - 対応: シングルカラム、タッチターゲット最小 44px を確保

### ネットワーク
12. **オフライン動作**: インターネット接続がない場合
    - 対応: LocalStorage のみで動作、外部 API 呼び出しなし

## 成功イメージ

実装が完了すると、以下のユーザーストーリーが実現します：

1. **朝のルーティン**: アプリを開き、昨日の習慣をチェック。連続日数が増える快感

4. **再開への抵抗感低減**: リセット後も「今日から再スタート」と前向きに操作できる UI

このシンプルながら実用的なツールを通じて、ユーザーの習慣化をサポートします。
