# シンプルな習慣トラッカー — Haiku-Opus パターン実装

## 概要

シンプルな習慣トラッカーは、毎日繰り返す習慣（運動、読書、瞑想など）を追跡し、連続実施日数をカウントするウェブアプリケーションです。Haiku（簡潔な設計思想）と Opus（豊かな実装能力）を組み合わせた実装です。

## 特徴

- **シンプルな UI**: 迷わない、シンプルな操作性
- **ローカルストレージ保存**: ブラウザを閉じてもデータが残る
- **リアルタイム統計**: 連続日数と最高記録を自動計算
- **レスポンシブ対応**: スマートフォンからデスクトップまで対応
- **外部ライブラリ不要**: 純粋な HTML/CSS/JavaScript で実装

## 使い方

### 起動方法

1. このフォルダを任意の場所に配置します
2. `index.html` をブラウザで開きます（ローカルファイルでOK、サーバー不要）
3. ブラウザの開発者ツールで localStorage が有効になっていることを確認してください

### 基本操作

#### 1. 習慣を追加する

1. 上部の「新しい習慣を追加」セクションで習慣名を入力
   - 例：「運動」「読書」「瞑想」など
2. 「追加」ボタンをクリック
3. 習慣一覧に新しい習慣が表示されます

**制限事項**：
- 最大 10 個まで登録可能
- 1 文字以上 30 文字以下
- 空文字やスペースのみの入力は不可
- 同じ名前の習慣は重複登録できません

#### 2. 習慣を実施する

1. 習慣一覧で「今日実施」ボタンをクリック
2. 本日の実施が記録されます
3. ボタンが「✓ 実施済み」に変わり、同日内での重複マークを防ぎます

**重要**：
- 同じ日に複数回クリックしても **1 回分のみ** カウントされます
- 翌日以降は再度「今日実施」ボタンで実施を記録できます

#### 3. 統計情報を確認する

- **連続日数**: 本日から遡って、連続して記録がある日数
  - 例：今日・昨日・一昨日に記録がある → 3 日
  - 例：今日はあるが昨日がない → 0 日にリセット
- **最高記録**: これまでの最長連続実施日数（更新されない）
- **総記録**: その習慣の総実施日数

#### 4. 習慣を削除する

1. 習慣一覧で「削除」ボタンをクリック
2. 確認ダイアログが表示されます
3. 「OK」を押すと習慣と全記録が削除されます

**注意**：
- 削除は確認後、その習慣と関連する全記録が失われます
- 取り消しはできませんので、確実に不要な習慣のみ削除してください

## 実装の詳細

### アーキテクチャ

本アプリケーションは 3 層で設計されています：

#### 1. Store（状態管理）

```javascript
Store = {
    getState(),           // 現在の状態を取得
    setState(state),      // 状態を保存
    addHabit(name),       // 習慣追加
    deleteHabit(habitId), // 習慣削除
    checkInToday(habitId),// 本日実施を記録
    getConsecutiveDays(), // 連続日数を計算
    getMaxConsecutiveDays() // 最高記録を取得
}
```

- LocalStorage を使用してデータを永続化
- JSON で習慣と記録を管理
- すべての状態変更は `setState()` を通じて行われる

#### 2. UI（画面表示）

```javascript
UI = {
    renderHabits(),     // 習慣一覧を描画
    renderStatistics(), // 統計情報を描画
    showMessage(),      // フォームメッセージを表示
    escapeHtml()        // XSS 対策
}
```

#### 3. Controller（イベント処理）

```javascript
Controller = {
    onAddHabit(e),      // 習慣追加フォーム処理
    onCheckIn(habitId), // チェックイン処理
    onDeleteHabit(habitId), // 削除処理
    refreshUI()         // 画面全体を更新
}
```

### 日付計算の仕組み

#### ローカルタイムゾーン対応

日付比較に `Date.toISOString()` ではなく、ローカル時刻を `YYYY-MM-DD` 形式で正規化：

```javascript
const today = new Date();
const dateStr = `${year}-${month}-${day}`;
```

これにより、ユーザーのタイムゾーンで「今日」が正確に判定されます。

#### 連続日数計算ロジック

1. 本日から過去へ遡り、毎日の記録をチェック
2. 記録がある限りカウント、ない日が出現したら終了
3. **重要**: 本日に記録がない場合、連続日数は 0

```javascript
例：
記録: 2025-05-07, 2025-05-06, 2025-05-05, 2025-05-03
本日: 2025-05-07 → 連続3日
本日: 2025-05-08 → 連続0日（本日記録なし）
```

#### 最高記録計算ロジック

1. すべての記録日付をソート
2. 隣同士の日付が 1 日差なら連続、それ以外でリセット
3. 最長の連続期間を記録

### エッジケース対応

| ケース | 対応 |
|--------|------|
| 同一日に複数チェックイン | `Array.includes()` で重複チェック |
| 習慣の重複登録 | `addHabit()` で事前バリデーション |
| 最大10個超過 | UI でアラート表示、追加を拒否 |
| 空文字や空白のみ入力 | `trim()` でチェック、追加を拒否 |
| 30字超の習慣名 | HTML 属性で `maxlength="30"` を指定 |
| LocalStorage 満杯 | `catch` で `QuotaExceededError` をハンドル |
| 不正な JSON データ | `try-catch` で parse エラーを処理し、初期化 |

## トラブルシューティング

### データが保存されない

1. ブラウザの設定を確認
   - プライベート/シークレットモードでは LocalStorage が機能しません
   - 通常モードで使用してください

2. LocalStorage をクリアしていないか確認
   - DevTools → Application → Local Storage から確認できます

3. ストレージ容量を確認
   - LocalStorage は通常 5-10 MB 制限
   - 習慣が少ないアプリなら問題ない容量です

### 「最大10個まで」警告が出ない

JavaScript コンソールを確認：
```javascript
// DevTools Console で実行
Store.getState().habits.length // 習慣数を表示
```

### 連続日数がリセットされた

以下の場合に連続日数は 0 にリセットされます：
- 本日に実施を記録していない
- 昨日以降、記録がない日がある

これは仕様です。翌日以降に「今日実施」をマークすると、連続カウントが再開されます。
