# テーマA: ウェブアプリ開発

## タイトル
「マークダウンライブプレビューエディター」

## 背景・目的
マークダウン形式でテキストを書いて、リアルタイムにHTMLプレビューを確認できるツールは多くの開発者に使われています。
このタスクでは、HTML/CSS/JavaScript のみを使った、シンプルで実用的なマークダウンエディターを作成してください。
外部の npm ライブラリに依存せず、ブラウザ単体で動作する実装を目指します。

## 想定ユーザー
- マークダウンで文書を執筆したい人
- プレビュー確認しながら執筆したい人
- ローカルで簡単に使えるツールを求める人

## 要求事項

### Must-Have（必須）
1. **マークダウン入力欄** — テキストエリアで マークダウン記法を入力できる
   - 受け入れ条件: テキストを入力可能で、改行が反映される
2. **リアルタイムプレビュー** — 入力中に右側（または下側）にHTMLプレビューが同期表示される
   - 受け入れ条件: 入力の約100ms以内にプレビューが更新される
3. **基本的なマークダウン対応** — 以下の要素を最低限実装
   - 見出し（H1～H6）
   - 段落、太字、イタリック
   - リスト（順序なし、順序付き）
   - コードブロック
   - リンク
   - 受け入れ条件: 上記全てがプレビューに正しく表示される
4. **ローカルストレージへの自動保存** — テキスト入力を自動保存し、ページリロード後も復元
   - 受け入れ条件: ブラウザを閉じて再度開いた時、テキストが残っている

### Should-Have（あると良い）
1. **テーマ切り替え** — ライト/ダークモードの切り替え
2. **入力フォント設定** — モノスペース/サンセリフ等、フォントの選択肢
3. **プレビューのCSSスタイリング** — 見やすいデフォルトスタイル（マージン、背景色等）
4. **印刷機能** — プレビューをPDFまたは紙で印刷できる

### Nice-to-Have（実装できれば）
1. **表（テーブル）対応** — マークダウンの表記法に対応
2. **コピーボタン** — HTMLをクリップボードにコピー
3. **リセットボタン** — テキストを初期化

## 技術制約
- **言語**: HTML, CSS, JavaScript のみ
- **外部ライブラリ**: 不可（マークダウンパーサーも自作するか、シンプルな正規表現を使う）
- **CDN**: 可（Google Fonts等、見た目を整えるために外部CSSライブラリは許可）
- **動作環境**: モダンなブラウザ（Chrome, Firefox, Safari, Edge の最新版）で動作すること

## 出力ファイル構成
```
output/
├── index.html      — メインファイル（HTMLとJavaScriptを含む）
├── style.css       — スタイルシート（外部CSSが必要な場合）
├── script.js       — JavaScript（外部化する場合）
└── README.md       — 使い方（ブラウザでの開き方、マークダウン記法の簡易説明）
```

## 評価の観点
- すべてのMust-Have要件が実装されているか
- マークダウン解析が正確か
- プレビューのUIが見やすく、使いやすいか
- コード品質（関数分割、命名の一貫性等）

## 完成イメージ
ブラウザで直接開いて、すぐに使えるマークダウンエディター。
保存や複雑な設定は不要で、「書いたら自動保存」「リアルタイムプレビュー」で気持ちよく執筆できるツール。
