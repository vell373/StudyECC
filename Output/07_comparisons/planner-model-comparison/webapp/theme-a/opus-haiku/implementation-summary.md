## 設計上の判断

### 1. マークダウンパーサーの実装方式
- **決定**: 正規表現ベースのシンプルな実装
- **理由**: CommonMark完全準拠は要件外。自作パーサーで要件の5種要素（見出し・強調・リスト・コード・リンク）をカバー

### 2. XSS対策
- **決定**: ユーザー入力をHTML エスケープしてから処理
- **実装**: `escapeHtml()` メソッドで `<`, `>`, `&`, `"`, `'` を HTML エンティティに変換
- **効果**: スクリプトインジェクション防止

### 3. コードブロック処理の工夫
- **決定**: コードブロック内ではマークダウン記法を処理しない
- **実装方式**: 
  1. 先にコードブロック（```...```）を抽出してプレースホルダに置換
  2. 他のマークダウン記法を処理
  3. 最後にプレースホルダをコードブロックで復元
- **効果**: ネストされた記号の干渉を防止

### 4. デバウンス戦略
- **プレビュー更新**: 50ms debounce
- **ストレージ保存**: 500ms debounce
- **ページ離脱時**: `beforeunload` イベントで保険の保存
- **理由**: 入力の高速化と CPU 負荷のバランス

### 5. レスポンシブ対応
- **広い画面（768px以上）**: CSS Grid で左右2ペイン配置
- **狭い画面（768px未満）**: メディアクエリで上下分割に自動切替
- **実装**: `@media (max-width: 768px)` で grid-template-columns を調整

### 6. テーマ管理
- **ライトモード**: デフォルトカラー（白背景・黒テキスト）
- **ダークモード**: CSS 変数（`--bg-color`, `--text-color` 等）で切替
- **永続化**: localStorage に `editor_theme` キーで保存
- **リセット**: ページロード時に localStorage から復元

### 7. フォント設定
- **選択肢**: Monospace (Roboto Mono) / Sans-serif (Roboto)
- **保存**: localStorage に `editor_font` キーで保存
- **反映**: CSS クラス切り替え（`.input-pane.font-mono`, `.input-pane.font-sans`）

## 既知の問題・制限

### 実装の制約
1. **完全なCommonMark準拠なし**: 要件外の記法（脚注・定義リストなど）は非対応
2. **深いネスト非対応**: リスト内リスト・リスト内強調などの深いネストは完璧ではない
3. **行末スペース2つの改行非対応**: マークダウンの「行末スペース2つで改行」は実装していない（空行で段落区切りのみサポート）
4. **テーブル未実装**: `| col1 | col2 |` 構文は非対応

### ブラウザ互換性
- **動作確認環境**: Chrome/Firefox/Safari/Edge の最新版を想定
- **プライベートブラウジング**: localStorage が無効な場合はメモリ内のみ保持

### パフォーマンス
- **大規模テキスト**: 数万字以上では debounce で遅延が感じられる可能性
- **複雑なマークダウン**: ネストが深い場合、パースが遅くなる可能性

### 使い方
1. **左パネルにマークダウン入力**: テキストエリアにマークダウン形式でテキストを入力
2. **右パネルにプレビュー表示**: 入力すると自動的に右側にHTMLプレビューが表示
3. **テーマ切替**: ヘッダーの 🌙 ボタンでダークモード/ライトモード切替
4. **フォント選択**: ヘッダーのドロップダウンメニューでフォント変更
5. **HTMLコピー**: 「Copy HTML」ボタンでプレビューのHTMLをクリップボードにコピー
6. **リセット**: 「Reset」ボタンでテキストを初期化（確認ダイアログあり）
