# 設計仕様書: マークダウンライブプレビューエディター

## 概要
ブラウザ単体で動作するシンプルかつ実用的なマークダウンエディターを構築する。左側に入力エリア、右側にHTMLプレビューを配置した2ペイン構成で、入力中にリアルタイム（100ms以内）にプレビューが同期更新される。マークダウンパーサーは外部ライブラリに依存せず、正規表現ベースで自作する。入力内容はlocalStorageへ自動保存され、ページをリロードしても復元される。「ブラウザでindex.htmlを開けば即座に使える」体験を最優先に設計する。

## 要件一覧

### Must-Have（実装必須）
1. **マークダウン入力欄**: `<textarea>` でマークダウンを入力でき、改行・インデントが保持される。タブキーでの入力にも違和感がないこと。
   - チェック方法: 複数行のテキストを入力し、改行が反映されることを目視確認。
2. **リアルタイムプレビュー**: 入力イベントから100ms以内にプレビュー側のHTMLが更新される（debounce 50〜80ms程度）。
   - チェック方法: DevToolsのPerformanceタブで input → DOM更新 のレイテンシを計測、または目視で「打鍵に追従する」感覚を確認。
3. **基本的なマークダウン対応**: 下記要素を自作パーサーで変換できる。
   - 見出し（`#`〜`######` → H1〜H6）
   - 段落（空行区切り）
   - 強調（`**bold**`、`*italic*`、`_italic_`）
   - リスト（`-`/`*` の順序なし、`1.` の順序付き）
   - インラインコード（`` `code` ``）とコードブロック（``` ``` ```）
   - リンク（`[text](url)`）
   - チェック方法: サンプル文書を入力して全要素が正しいHTMLタグに変換されることを確認。
4. **ローカルストレージ自動保存**: 入力後 500ms 以内に localStorage へ保存。リロード/ブラウザ再起動後に同じ内容が復元される。
   - チェック方法: 文章を入力 → タブを閉じる → 再度開く → テキストが残っていること。

### Should-Have（あると良い）
1. **ライト/ダークモード切替**: ヘッダーのトグルボタンで切替。設定は localStorage に保存。
2. **プレビューCSSスタイリング**: 見出し・コード・引用などに見やすいデフォルトCSS（適切なマージン、フォント、背景色）。
3. **入力フォント設定**: 入力エリアのフォントをモノスペース/サンセリフから選択可能（プルダウン）。

### Nice-to-Have（実装できれば）
1. **表（テーブル）対応**: `| col1 | col2 |` のマークダウン表記に対応。
2. **HTMLコピーボタン**: プレビュー側のHTMLをクリップボードにコピー。
3. **リセットボタン**: 確認ダイアログ後にテキストを初期化。
4. **印刷機能**: ブラウザの印刷ダイアログでプレビューのみを印刷（`@media print` でエディター部を非表示）。

## 技術スタック（推奨）
- **HTML5**: セマンティック要素（header/main/section）で2ペインレイアウトを構築。理由: 軽量・追加依存なし。
- **CSS3 (Grid + CSS変数)**: 2ペイン構成は CSS Grid で1行2列。テーマカラーは `:root` の CSS変数で定義しダークモード切替を簡素化。理由: ライブラリ不要でテーマ実装が容易。
- **Vanilla JavaScript (ES2020+)**: モジュール分割は `<script type="module">` で。理由: 制約により外部ライブラリ禁止のため。
- **正規表現ベースのパーサー**: `String.prototype.replace` を順次適用するシンプルな実装。理由: 完全なCommonMark準拠は不要、要件にある要素のみカバーすればよい。
- **localStorage API**: 自動保存に使用。理由: サーバーレスでシンプル。
- **Google Fonts (CDN)**: 任意でフォント調達のみCDN許可。理由: 視認性向上のため。

## 設計上の重要な判断点
- **マークダウンパーサーの実装方式**: 完全なCommonMark準拠は要件外なので、正規表現を順次適用する素朴な実装とする。理由: 自作の妥当な範囲に収め、保守性を担保。
- **XSS対策**: ユーザー入力をそのまま `innerHTML` に流すのは危険。マークダウン変換前に `<`, `>`, `&` をHTMLエスケープし、その後マークダウン記法のみを許可タグに置換する。理由: ローカルツールでも安全側に倒す。
- **debounce戦略**: 入力イベントごとにパースすると重い可能性があるため、50〜80ms の debounce を入れる。100ms以内更新の要件は満たす。
- **保存タイミング**: 入力ごとに毎回 localStorage に書くと負荷が出るため、500ms の debounce で保存。ページ離脱時 (`beforeunload`) にも保険で保存。
- **2ペインのレスポンシブ対応**: 幅が狭い画面（768px以下）では上下分割に切り替える（CSS Grid のメディアクエリ）。
- **コードブロック内のマークダウン記法**: コードブロック内では他のマークダウン変換を抑制する（先にコードブロックを抽出してプレースホルダ化し、最後に戻す）。

## 考慮すべきエッジケース
- **空入力**: プレビューも空にする（エラーを出さない）。
- **巨大テキスト（数万字）**: パースが遅くなる可能性があるため、debounce で吸収。完成後にパフォーマンス確認。
- **localStorage 容量超過**: try/catch でエラーを補足し、コンソールに警告を出す（ユーザー体験を壊さない）。
- **プライベートブラウジングで localStorage が無効**: `try/catch` で書き込み試行し、失敗時はメモリ内のみ保持。
- **HTMLタグを含む入力**: `<script>` などのタグはエスケープ後に表示する（XSS対策）。
- **不正なマークダウン**: 閉じていない `**` や `[`などは原文のまま表示（壊れない）。
- **ネスト要素**: リスト内の強調などの基本的なネストは変換するが、深いネスト（リスト内リストなど）は完璧でなくてよい。
- **改行の扱い**: マークダウンの「行末スペース2つで改行」は実装しなくてよい。空行で段落区切りのみサポート。
