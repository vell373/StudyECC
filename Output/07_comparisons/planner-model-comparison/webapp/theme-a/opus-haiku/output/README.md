# Markdown Live Preview Editor

ブラウザ単体で動作するシンプルなマークダウンエディター。左側に入力、右側にリアルタイムプレビューを表示する2ペイン構成。外部ライブラリに依存しません。

## 機能

### Must-Have（必須機能）
- **リアルタイムプレビュー**: マークダウン入力から 100ms 以内にプレビューが更新
- **マークダウン対応**: 見出し、段落、強調、リスト、インラインコード、コードブロック、リンク
- **自動保存**: 入力内容を localStorage へ自動保存。ページリロード後も復元
- **XSS対策**: ユーザー入力を安全にエスケープして処理

### Should-Have（便利機能）
- **ダークモード**: ヘッダーのアイコンで切り替え可能。設定は localStorage に保存
- **フォント選択**: モノスペース / サンセリフから選択可能
- **見やすいスタイリング**: コード・見出し・リストに適切なデフォルト CSS

### Nice-to-Have（オプション機能）
- **HTML コピー**: プレビューのHTML を クリップボードにコピー
- **リセットボタン**: エディター内容をクリア（確認ダイアログ付き）
- **レスポンシブ対応**: 幅 768px 以下では上下分割に自動切り替え
- **印刷対応**: ブラウザの印刷機能でプレビューのみを出力

## 使い方

### 起動

1. このフォルダの `index.html` をブラウザで開く（ダブルクリック推奨）
2. 左側のテキストエリアにマークダウンを入力
3. 右側に HTML プレビューがリアルタイム表示される

**注**: サーバー不要。ローカルファイルとして開くだけで使用可能です。

### 操作方法

| 操作 | 説明 |
|------|------|
| 左ペインに入力 | マークダウンを入力。改行は Shift+Enter or 通常のEnter で処理 |
| Tab キー | インデント（2スペース）を挿入 |
| 🌙 / ☀️ ボタン | ダークモード / ライトモード を切り替え |
| フォントセレクト | モノスペース / サンセリフ を選択 |
| Copy HTML | プレビューの HTML をクリップボードにコピー |
| Reset | エディターをクリア（確認ダイアログあり） |

### データ保存

- **自動保存**: 入力後 500ms 以内に localStorage へ保存
- **復元**: ページをリロードすると、前回の入力内容が復元される
- **プライベートブラウジング**: localStorage が無効な環境でも、セッション中はメモリに保持

## マークダウン対応記法

### 見出し

```markdown
# 見出し 1
## 見出し 2
### 見出し 3
#### 見出し 4
##### 見出し 5
###### 見出し 6
```

### 段落

空行で区切ることで段落になります。

```markdown
これは段落1です。

これは段落2です。
```

### 強調

```markdown
**太字** または __太字__
*斜体* または _斜体_
```

### リスト（順序なし）

```markdown
- 項目1
- 項目2
- 項目3
```

または

```markdown
* 項目1
* 項目2
```

### リスト（順序付き）

```markdown
1. 項目1
2. 項目2
3. 項目3
```

### インラインコード

```markdown
これは `inline code` です。
```

### コードブロック

```markdown
```
function hello() {
  console.log("Hello, World!");
}
```
```

### リンク

```markdown
[リンクテキスト](https://example.com)
```

### 水平線

```markdown
---
```

または

```markdown
***
```

または

```markdown
___
```

## 技術仕様

### マークダウンパーサー

- **実装方式**: 正規表現ベースの自作パーサー
- **外部ライブラリ**: なし（Vanilla JavaScript のみ）
- **対応記法**: 基本的なマークダウン（見出し・強調・リスト・コード・リンク）
- **コードブロック保護**: コードブロック内ではマークダウン記法を処理しない

### パフォーマンス最適化

- **debounce**: 入力イベントから 80ms の debounce でパース処理を抑制
- **localStorage の debounce**: 入力後 500ms でまとめて保存
- **コードブロック抽出**: 先にコードブロックを抽出してプレースホルダ化し、マークダウン記法の誤変換を防止

### XSS 対策

1. ユーザー入力をエスケープ（`<`, `>`, `&`, `"`, `'`）
2. マークダウン記法のみを HTML タグに置換
3. クリップボード操作では `navigator.clipboard.writeText()` を使用

### ローカルストレージ

- **保存内容**:
  - `markdown_content`: エディター入力テキスト
  - `editor_theme`: テーマ設定（light / dark）
  - `editor_font`: フォント設定（mono / sans）
- **容量上限**: 約 5-10 MB（ブラウザ依存）。超過時は コンソールに警告出力

### レスポンシブデザイン

- **通常表示（768px 以上）**: 2ペイン（左右分割）
- **モバイル表示（768px 以下）**: 上下分割。入力エリアが上 40%、プレビューが下 60%

## 既知の制限事項

- **CommonMark 非準拠**: 要件に必要な基本要素のみ対応
- **テーブル**: マークダウン形式のテーブルは未対応
- **脚注**: マークダウン脚注は未対応
- **行末スペース改行**: `行末スペース2つで改行`はサポートしていません。空行で段落区切りのみ
- **ネスト要素**: リスト内のリストなど深いネストは未対応

## トラブルシューティング

### localStorage が動作しない

**症状**: ページをリロードしてもテキストが復元されない

**原因**: プライベートブラウジングモード、ブラウザ設定、容量超過など

**対策**:
1. プライベートブラウジングを終了して再度開く
2. ブラウザの設定でローカルストレージを許可する
3. 不要なデータを削除して容量を確保する

### プレビューが更新されない

**症状**: マークダウン入力しても右側が反応しない

**原因**: JavaScript が無効になっている可能性

**対策**: ブラウザの開発者ツール（F12）で JavaScript が有効か確認

### スタイルが反映されない

**症状**: 色やレイアウトが崩れている

**原因**: CSS ファイル（style.css）が見つからない

**対策**: `index.html` と同じフォルダに `style.css` があることを確認

## 開発メモ

### コード構成

- **storage オブジェクト**: localStorage の アクセスを統一管理
- **MarkdownParser クラス**: マークダウン → HTML の変換ロジック
- **EditorApp クラス**: UI イベント処理、状態管理

### 拡張ポイント

1. **マークダウン記法追加**: `MarkdownParser.parse()` 内に新規の `convert~()` メソッドを追加
2. **新しい UI 機能**: `EditorApp` に新規メソッドを追加してイベントリスナーを登録
3. **スタイルカスタマイズ**: CSS 変数（`:root`）を編集してテーマを変更

### デバッグ

ブラウザの開発者コンソール（F12）で以下を実行して状態を確認できます：

```javascript
// 保存されたマークダウンを確認
localStorage.getItem("markdown_content");

// テーマ設定を確認
localStorage.getItem("editor_theme");

// localStorage をすべてクリア（注意: 入力内容も消えます）
localStorage.clear();
```
