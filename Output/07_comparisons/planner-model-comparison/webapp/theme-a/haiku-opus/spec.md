# 設計仕様書: マークダウンライブプレビューエディター

## 概要
HTML/CSS/JavaScript のみを使い、ブラウザで直接動作するマークダウンエディターを開発します。ユーザーはテキストエリアにマークダウン記法を入力しながら、リアルタイムにHTMLプレビューを右側（またはレスポンシブで下側）に確認できます。外部ライブラリに依存せず、入力は自動的にローカルストレージに保存されるため、ページをリロードしても作業内容が失われません。シンプルで実用的な執筆環境を提供し、マークダウン初心者から実務利用者まで幅広く活用できるツールです。

## 要件一覧

### Must-Have（実装必須）
1. **マークダウン入力フィールド**: TextAreaで改行を含むマークダウン記法を入力可能にする
   - 受け入れ条件: TextAreaに入力したテキストがそのままストレージに保存でき、改行が正しく反映される

2. **リアルタイムプレビュー表示**: 入力の約100ms以内にHTMLプレビューが同期更新される
   - 受け入れ条件: 3文字以上入力した段階でプレビューが反応開始し、デバウンス処理により入力完了から100ms以内に最終プレビューが表示される

3. **基本的なマークダウン解析**: 以下の要素を正規表現による自作パーサーで実装する
   - 見出し（# H1～### H6）
   - 段落（空行で分離）
   - 太字（**text**）、イタリック（*text*）
   - 順序なしリスト（- 項目）
   - 順序付きリスト（1. 項目）
   - インラインコード（\`code\`）
   - コードブロック（```lang\ncode\n```）
   - リンク（[text](url)）
   - 受け入れ条件: 全要素がプレビューに正しくHTMLに変換され表示される、スタイルが適用される

4. **ローカルストレージへの自動保存**: テキスト変更時に自動保存、ページリロード時に復元
   - 受け入れ条件: DevTools→Application→LocalStorageで値が確認でき、ブラウザ再起動後も同じテキストが表示される

### Should-Have（あると良い）
1. **ライト/ダークモード切り替え**: UIトグルボタンで全体のテーマを変更
   - 受け入れ条件: ライトモードとダークモードが視覚的に区別でき、設定がローカルストレージに保存される

2. **入力エリアのフォント選択**: セレクトボックスでモノスペース/サンセリフ等を選択可能にする
   - 受け入れ条件: Google Fonts から最低2種類のフォントをCDNで読み込み、選択すると即座に反映される

3. **プレビューエリアのスタイリング**: 見やすいデフォルトスタイル（マージン、背景色、行高等）を適用
   - 受け入れ条件: h1～h6のサイズが段階的、段落のマージンが適切、全体的に可読性が高い状態

### Nice-to-Have（実装できれば）
1. **テーブル対応**: マークダウンのテーブル記法（\| col1 \| col2 \| 形式）に対応
2. **HTMLコピーボタン**: 生成されたHTMLをクリップボードにコピー
3. **リセットボタン**: 確認ダイアログ付きでテキストを初期化

## 技術スタック（推奨）
- **HTML5**: セマンティックHTML、flexbox/grid レイアウト
  - 選んだ理由: ブラウザ標準機能で、シンプルで高速な構造実現が可能
- **CSS3**: カスタムプロパティ（CSS Variables）でテーマ切り替えを実装
  - 選んだ理由: ダークモード対応が容易で、サイズが小さく保守性が高い
- **Vanilla JavaScript**: 正規表現ベースのマークダウンパーサー、デバウンス処理
  - 選んだ理由: 外部ライブラリなしで十分実装可能、本来の動作をシンプルに保証
- **Google Fonts CDN**: フォント選択肢を充実（Courier Prime, Lato 等）
  - 選んだ理由: CDN使用許可範囲内で、ダウンロード不要かつ軽量
- **LocalStorage API**: テキストの自動保存と復元
  - 選んだ理由: ブラウザ標準で、サーバー不要

## 実装フェーズ計画

### Phase 1: コア機能実装 — マークダウン解析とプレビュー基盤確立
**目標**: 基本的なマークダウン解析とリアルタイムプレビューが動作する状態

**対象要件**: Must-Have 1～4

**作業内容**:
1. HTML 構造の構築（TextArea、プレビューエリア、基本的なレイアウト）
2. 正規表現ベースのマークダウンパーサー実装
   - 見出し、段落、太字、イタリック、リスト、コードブロック、リンク
3. EventListener（input, change）によるリアルタイムプレビュー更新
4. デバウンス処理（100ms遅延）で過度な再描画を防止
5. LocalStorage への自動保存（save()関数）と復元（load()関数）

**完了条件**:
- [ ] TextAreaへの入力が即座にLocalStorageに保存される
- [ ] プレビューが100ms以内に更新される
- [ ] マークダウン要素（見出し、リスト、コード、リンク）が正しくHTML化される
- [ ] ページリロード後、TextAreaにテキストが復元される
- [ ] ブラウザコンソールにエラーが出ていない

### Phase 2: UI/UX 拡張と完成度向上 — Should-Have実装
**目標**: 使いやすさと見栄えを向上させ、実用的な完成度に到達

**対象要件**: Should-Have 1～3、Nice-to-Have 1

**作業内容**:
1. CSSカスタムプロパティ（--color-bg, --color-text等）でテーマ定義
2. トグルボタンでダークモード切り替え、LocalStorageに保存
3. Google Fonts 統合、フォント選択セレクト実装
4. プレビューのCSS強化（h1～h6のサイズ階層、段落マージン、行高、コードブロックのスタイル）
5. テーブル対応（| col | col | 形式）を正規表現で追加
6. HTMLコピーボタンの実装（Clipboard API使用）

**完了条件**:
- [ ] ダークモード/ライトモードの切り替えが機能し、設定が保存される
- [ ] フォント選択で複数のフォントが表示可能かつ即座に反映される
- [ ] プレビューのスタイルが見やすく、ドキュメント品質に見える
- [ ] テーブルが正しくHTML化される（TH/TD要素）
- [ ] HTMLコピーボタンが正常に機能する
- [ ] すべてのMust-Have要件が依然として動作している

## 品質基準（これが揃えば「完成」とみなす）
- **機能的正確性**: Must-Have全4項目が完全に実装され、テストケース（見出し、リスト、コード、リンク等）に対して正しいHTMLを出力する
- **パフォーマンス**: プレビュー更新が100ms以内に完了する、LocalStorage保存がブロッキングしない
- **UI/UX**: 入力フィールドとプレビューが明確に分離、フォント・テーマが簡単に変更可能、エラーメッセージなく動作する
- **コード品質**: 関数が50行以下に分割、変数名が明確（parseMarkdown, renderPreview, debounce等）、コメントで「なぜ」を説明している
- **ブラウザ互換性**: Chrome/Firefox/Safari/Edgeの最新版で動作を確認、LocalStorage が正常に機能している
- **ドキュメント**: README.md にマークダウン記法の対応一覧、基本的な使い方、ローカル開き方が記載されている

## 設計上の重要な判断点

- **マークダウンパーサーの実装方式**: 正規表現ベースの逐次処理を選択
  - 判断理由: 単純明快で保守性が高く、外部ライブラリなしで実現可能。複雑な解析が必要な場合は段階的に対応
  - 注意: ネストされた要素（太字の中のイタリック等）の対応は優先度を下げ、初版では無視する

- **デバウンス戦略**: 入力時に100msのデバウンスを適用
  - 判断理由: 入力が停止してから実際にパース処理をトリガーするため、CPUスパイク回避と応答性のバランスを取る
  - 注意: 完全なリアルタイム性よりもパフォーマンスを優先

- **LocalStorage 上限**: 1文書あたり最大5MB程度と想定
  - 判断理由: 一般的なブラウザのLocalStorage上限は5～10MBで、通常の執筆用途では十分。超過時はユーザーに警告
  - 注意: 複数文書の管理は初版スコープ外

- **テーマシステム**: CSS カスタムプロパティで全色を管理
  - 判断理由: ライト/ダークモード切り替えが簡潔になり、将来的な拡張も容易
  - 注意: iOS Safari 等の古いブラウザでは未対応だが、指定の最新版対応で問題なし

## 考慮すべきエッジケース

- **空のテキスト入力**: プレビューが空欄に表示される（エラーではなく正常）
  - 対応方針: プレビューエリアに「プレビューがここに表示されます」というプレイスホルダーテキストを表示

- **特殊文字とエスケープ**: マークダウンメタキャラクター（\*, \#, \[ 等）が入力されるケース
  - 対応方針: 正規表現で適切にエスケープ処理を行い、\`code\` はそのまま表示（HTMLエスケープ）

- **HTMLコンテンツのインジェクション**: ユーザーが\<script\>タグを入力した場合
  - 対応方針: プレビューエリアに innerHTML ではなく textContent で表示するか、DOMParser で安全にパース

- **LocalStorage の未対応環境**: プライベートモード等でStorage API が利用不可なケース
  - 対応方針: try-catch で保存処理をラップし、失敗時はコンソール警告（サイレント失敗）

- **非常に大きなテキスト入力**: 1MBを超えるマークダウンが入力されるケース
  - 対応方針: 初版では警告なし、将来的にページ遅延通知を追加検討

- **同時ウィンドウ編集**: 複数タブで同じページを開く場合のLocalStorage競合
  - 対応方針: 各タブは独立して動作し、最後の保存が優先される（シンプル戦略）

- **改行やインデント**: コードブロック内の改行とインデントが正確に保持されているか
  - 対応方針: \<pre\>\<code\> で囲み、スペースを HTML エンティティ (&nbsp;) に変換

## 制約・注意事項

- **外部ライブラリ禁止**: marked.js 等のマークダウンパーサーライブラリは使用できない。正規表現と文字列操作のみで実装
- **CDN 利用制限**: Google Fonts 等の見た目向上に関する外部CSSは許可されるが、JavaScript ライブラリは禁止
- **ブラウザ互換性**: 指定の最新版（Chrome, Firefox, Safari, Edge）での動作保証が必須。IE11 等の旧版は非対象
- **ファイル構成**: index.html（HTML+JavaScript）、style.css、script.js（分割可）、README.md（説明書）の4ファイル（または3ファイル）での納品
- **パフォーマンス要件**: 通常の使用範囲（数千行以下）で、プレビュー更新が目視できるラグなく完了すること
- **セキュリティ**: ユーザー入力は必ず HTML エスケープ処理し、XSS 対策を講じる。スクリプトタグの直接実行は禁止
- **LocalStorage 使用**: ブラウザの開発者ツール（DevTools）で内容が確認可能であること。保存容量警告がない場合、デフォルト保存動作を続行
