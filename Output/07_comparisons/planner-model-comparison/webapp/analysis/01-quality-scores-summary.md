## 4. パターン別特性サマリー

### Opus-Haiku（高品質計画 + 低コスト実装）

**全テーマ平均スコア: 84.67点（A-）**

**総合強み**:
- Theme B（ポモドーロ）で最高スコア93.75点を獲得（S評価）
- コード規模が比較的小さい傾向（Theme B: 1,131行、Theme C: 571行）
- Theme C（習慣トラッカー）で要件充足100点達成
- XSS対策・beforeunload保険保存など、セキュリティへの配慮が堅牢

**総合弱み**:
- Theme A（マークダウン）で正規表現バグリスク・テーブル非対応
- アクセシビリティ（aria要素）が比較的限定的な傾向
- 初回UIでデフォルトテキストが欠落（学習曲線の増加）
- 創造性スコアが全テーマで相対的に低い傾向

**評価**: B+〜A範囲（テーマ依存性高い。ポモドーロで優秀、マークダウンで課題）

---

### Haiku-Opus（低コスト計画 + 高品質実装）

**全テーマ平均スコア: 90.50点（A）**

**総合強み**:
- 全テーマで完全優位またはほぼ同点（3テーマ中3テーマ勝利）
- 平均90.5点で安定した高品質を実現
- Theme A（マークダウン）で90.25点と大幅優位（+16点）
- アクセシビリティ・ユーザーフィードバック・ドキュメントが一貫して充実
- 保守性を重視した設計（Theme C: 450行でシンプル）

**総合弱み**:
- Theme B で全体的にコード量が多い傾向（CSS 673行）

- 印刷対応（@media print）が実装されていない（Theme A・C）
- Theme C で要件充足が90点（Should-Have一部未実装）

**評価**: A（実務推奨。ユーザー中心の設計が一貫）

## 7. 総合結論

### ウェブアプリ開発での推奨事項

**公開アプリ・ユーザー向けアプリ**  
→ **Haiku-Opus 推奨**
- 理由: アクセシビリティ・ドキュメント・UXフィードバック一貫
- 平均90.5点で安定した高品質
- Theme Aのような複雑仕様でも設計の一貫性維持

**学習目的・内部ツール・プロトタイプ**  
→ **テーマ依存**
- Theme B（ポモドーロ）は両パターン優秀（93.75 vs 94.25）、選択自由
- Theme C（習慣トラッカー）は両パターン優秀（86 vs 87）、選択自由
- Theme Aのような複雑仕様の場合はHaiku-Opus推奨

### パターン選択のフローチャート

```
Q1. アプリの仕様複雑度は？
├─ 高（マークダウン編集・複雑な変換）→ Haiku-Opus 強推奨
├─ 中（タイマー・UI複雑度中程度）→ どちらでも可（わずかHaiku-Opus優位）
└─ 低（CRUD・シンプル統計）→ どちらでも可

Q2. ユーザー向けの公開予定は？
├─ Yes → Haiku-Opus（アクセシビリティ充実）
└─ No → Opus-Haiku でも可（コスト効率重視なら）

Q3. 保守性・拡張性を重視するか？
├─ Yes → Haiku-Opus（設計の一貫性・小さいコード規模）
└─ No → Opus-Haiku でも可（完成度が高ければ）
```
