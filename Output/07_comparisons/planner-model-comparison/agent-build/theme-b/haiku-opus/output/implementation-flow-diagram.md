## エッジケース処理フロー

```
Input Diff
    │
    ├─→ [複数ファイル変更]
    │   └─→ ファイル別に分割処理
    │       └─→ 各ファイルの属性を判定
    │           └─→ マージ時に「File: XXX」タグで区別
    │
    ├─→ [大規模 diff (>1000 行)]
    │   └─→ 警告フラグ 🚨 自動化度低下
    │       └─→ 「要手動確認」レポート
    │
    ├─→ [複数属性混在]
    │   (新機能 + バグ修正 + セキュリティ)
    │   └─→ Major/Minor/Patch を組み合わせ
    │       └─→ Major 優先（バージョン判定）
    │           └─→ セクション分割記載
    │
    ├─→ [既存ドキュメント形式異なる]
    │   └─→ テンプレート検証エラー
    │       └─→ 「[要記載]」プレースホルダで提案
    │
    ├─→ [矛盾多数（>3 件）]
    │   └─→ 🔴 要レビュー フラグ
    │       └─→ 自動化比率を表示
    │
    ├─→ [非公開関数変更 (_prefix)]
    │   └─→ スキップ（API Reference 外）
    │       └─→ CHANGELOG にのみ記載（内部向け）
    │
    └─→ [多言語混在ドキュメント]
        └─→ Phase 2 機能
            └─→ 言語自動判定・分別処理

```
