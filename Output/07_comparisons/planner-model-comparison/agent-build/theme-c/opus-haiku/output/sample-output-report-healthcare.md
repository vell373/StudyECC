---

## Executive Summary

推奨テック スタック: **Java + Spring Security + PostgreSQL + AWS GovCloud**

本プロジェクト（医療診断支援 AI）は、HIPAA・GDPR・PCI-DSS 準拠が絶対条件であり、**セキュリティ（9/10）** が圧倒的に優先です。患者データの完全な保護・監査ログ、医療フレームワークの豊富な実装があるため、**Java + Spring Security** が最適と判断しました。開発期間が 12ヶ月あることも、Java の堅牢性を活用できる利点です。

総合スコア: **9.1/10**（Java）vs 8.7/10（Go + Gin）vs 7.2/10（Node.js）

## 候補テック スタック

### 1. Java + Spring Security + PostgreSQL + AWS GovCloud（推奨）

**選定理由**: HIPAA・GDPR・PCI-DSS 準拠フレームワーク最充実。医療システムの実績豊富。

**メリット**:
- Spring Security: 認証・認可・CSRF保護・SQL インジェクション対策が標準実装
- HIPAA 対応: 監査ログフレームワーク（Spring Actuator），トランザクション記録が容易
- 暗号化: Bouncy Castle, Keycloak 等により AES-256、TLS 1.3 実装容易
- AWS GovCloud: HIPAA/FedRAMP 準拠、隔離されたクラウド環境で PHI 保護
- チーム: Java 経験者 3名で即座に対応可能

**課題とその対応**:
- 開発速度: セットアップ複雑だが 12ヶ月期限で許容（セキュリティが優先）
- AI 推論統合: TensorFlow Serving（REST API）で Python・Go 推論エンジンと連携
- リソース効率: Spring Boot アプリケーション 512MB RAM × 複数インスタンス → 予算対応（医療向けで許容）

---

### 2. Go + Gin + PostgreSQL + AWS

**選定理由**: セキュリティと軽量性のバランス。Go の並行処理で高パフォーマンス。

**メリット**:
- シンプルな言語設計で セキュリティ脆弱性少ない
- 軽量（メモリ効率良好）で医療施設のローカル環境にも対応
- 暗号化・TLS 実装が標準ライブラリで充実
- パフォーマンス: AI 推論レイテンシ < 100ms 達成容易

**課題とその対応**:
- セキュリティフレームワーク薄い: Gin は HTTP フレームワークのみ → 認証・監査ログを自実装（実装工数 +6週間）
- HIPAA コンプライアンスツール: Spring Security 比で実装が浅い
- チーム採用: Go エンジニア少ないため、人材確保が困難
- 医療システム実績: Java 比で実績少ない（リスク）

---

### 3. Rust + Actix + PostgreSQL + AWS

**選定理由**: メモリ安全性最高峰。所有権システムでバッファオーバーフロー等を根本排除。

**メリット**:
- メモリ安全性: 所有権・ボロー制度で実行時メモリ脆弱性を根本排除（セキュリティ 10/10）
- パフォーマンス: 最高峰の処理速度・メモリ効率で医療施設での効率的運用
- 低レイテンシ: AI 推論統合で 50ms 以下も可能

**課題とその対応**:
- 学習曲線: Rust の習得に 3-6ヶ月必須（チーム全員未経験であれば +6ヶ月開発遅延）
- HIPAA フレームワーク: コミュニティが Java・Go 比で小さく、実装例が少ない
- チーム採用: Rust エンジニアは極めて少ないため、採用困難
- 医療システム実績: ほぼなし（信頼性の実績が必須な医療分野では非推奨）

**結論**: パフォーマンス・セキュリティ面では最優秀だが、実装体制・人員確保・実績がない。医療用途では採用できない。

## 最終推奨

### 推奨テック スタック

| 項目 | 選択 |
|------|------|
| **言語** | Java 17 (LTS) |
| **フレームワーク** | Spring Boot 3.0 + Spring Security 6.0 |
| **セキュリティ** | Keycloak (SSO) + Spring Data Encryption |
| **データベース** | PostgreSQL 15 + Transparent Data Encryption (TDE) |
| **クラウド** | AWS GovCloud（HIPAA/FedRAMP 準拠） |
| **監査ログ** | Spring Actuator + AWS CloudTrail |

### 選定根拠

1. **セキュリティ（最優先軸）への適合度**:
   - セキュリティスコア 10/10 × 重み 0.5 = 5.0
   - パフォーマンス 8/10 × 重み 0.3 = 2.4
   - スケーラビリティ 9/10 × 重み 0.15 = 1.35
   - 開発速度 6/10 × 重み 0.05 = 0.3
   - **合計: 9.05/10**

2. **医療規制への完全準拠**:
   - HIPAA: 監査ログ 6年保管（Spring Actuator + CloudTrail で対応）
   - GDPR: データ削除権（Right to be forgotten）実装容易
   - PCI-DSS: Spring Security で決済情報暗号化
   - AWS GovCloud: 隔離クラウド環境で PHI 保護

3. **チーム状況への適合**:
   - Java 経験者 3名で即座に対応可能
   - セキュリティ専門家 1名で Spring Security カスタマイズ監督可能
   - 開発期間 12ヶ月で十分な検証・テスト期間確保

4. **リスク評価と対応**:
   - 規制変更リスク → Spring Security のアップデート対応が容易（20年超の実績）
   - セキュリティ脆弱性検出時 → 迅速なセキュリティパッチが利用可能
   - スケーラビリティリスク → 複数施設対応時にマイクロサービス化容易（Spring Cloud）

## 実装計画（12ヶ月）

### Phase 1: セキュリティベース設計（0-2ヶ月）
