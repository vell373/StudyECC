# プロジェクト要件書：スマートホーム IoT 管理プラットフォーム

**作成日**: 2026-05-09  
**プロジェクト名**: SmartHome Hub Central  
**事業領域**: IoT（スマートホーム・エッジコンピューティング）

---

## 1. プロジェクト概要

### プロジェクト名
SmartHome Hub Central - デバイス・センサー一括管理プラットフォーム

### ビジネス目標
- 複数の IoT デバイス（温度・湿度・照度・ドアロック等）を統一インターフェースで管理
- エッジコンピューティング対応（ローカル処理・オフライン動作）
- 低消費電力・低遅延を実現

---

## 2. 機能要件

### コア機能
- 複数デバイスの登録・管理
- リアルタイムセンサーデータ受信（100ms 以下）
- ローカルネットワーク内でのデバイス制御（オフライン対応）
- クラウド同期（ネットワーク復帰時）
- 自動化ルール定義（例: 温度が 25°C 以上でエアコン起動）

### スケール要件
- **デバイス数**: 初期 1世帯あたり 20-50デバイス、全国で 10,000世帯 → 500,000デバイス
- **QPS**: 各デバイスからの送信頻度 1秒ごと → 500,000 QPS
- **低遅延**: センサーから制御コマンド実行まで < 100ms
- **データ保持**: 直近 30日のセンサーデータをローカル保存

---

## 3. 非機能要件

### パフォーマンス
- **必須度**: **最高優先度**
- **要件**:
  - センサー読み取り遅延: < 10ms
  - ネットワーク遅延: < 50ms
  - 制御コマンド実行: < 100ms（合計）
  - CPU 使用率: < 30% on 1GHz processor
  - メモリ: < 128MB RAM

### セキュリティ
- **必須度**: 高
- **要件**:
  - デバイス間通信: TLS 1.2以上 + 暗号化
  - 認証: デバイス証明書ベース
  - 暗号化キー管理: ローカルキーストア + クラウド同期

### スケーラビリティ
- **必須度**: 中～高
- **要件**: 水平スケーリング対応、メッセージキュー活用（MQTT/AMQP）

### 可用性
- **SLA**: 99.5% アップタイム（クラウド側）
- **オフライン対応**: ローカルハブは完全オフライン動作可能

---

## 4. チーム・制約条件

### チーム構成
- **規模**: 4名
  - エッジ / 組込エンジニア: 2名（C, C++経験）
  - バックエンド: 1名
  - DevOps: 1名
- **既存スキル**: C 経験が強い、組込開発歴長い

### リリース計画
- **期限**: 9ヶ月で初回出荷
- **段階**: PoC(2ヶ月) → Beta Hardware(3ヶ月) → GA(4ヶ月)

### 予算・インフラ
- **開発予算**: $800K
- **ハードウェア**: Raspberry Pi / ARM ベースの組込デバイス
- **クラウド**: AWS IoT Core / Azure IoT Hub 検討中

### 既存資産
- C で書かれた 3つのドライバ・ファームウェア
- MQTT クライアントライブラリ（既製）

### 商用サポート
- **必須**: MQTT ブローカー・センサーライブラリのサポート体制

---

## 5. 技術的制約

### エッジデバイス制約
- **プロセッサ**: ARM Cortex-A (1-4 GHz)
- **メモリ**: 512MB - 2GB RAM
- **ストレージ**: 4-16GB SD カード
- **電力消費**: バッテリー駆動の要件なし（AC 電源）

### ネットワーク
- **バックホール**: Wi-Fi 802.11ac + Ethernet
- **デバイス通信**: Zigbee / BLE / Wi-Fi (選択)
- **オフライン許容**: 24時間以上のネットワーク遮断でも動作

### リアルタイム性
- MQTT 遅延許容: < 100ms
- GPIO レスポンス: < 10ms

---

## 6. 非技術的制約

### 市場要件
- ユーザーが技術者でない（セットアップ簡単に）
- 複数言語対応予定（当初は英語・日本語）

### 規制
- 電波法（各国）への準拠（周波数・出力等）
- 一般消費者製品として安全基準対応
