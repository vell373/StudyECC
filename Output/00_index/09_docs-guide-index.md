# ECC ドキュメント・ガイド索引

ECCには、アーキテクチャの深い理解や、特定の状況でのベストプラクティスを解説した膨大なドキュメントが存在します。

---

## 📚 主要なガイド (The Guides)
ECCを使いこなすための、目的別の3大ガイドです。

| ガイド名 | 内容・目的 |
|---------|-----------|
| **The Shortform Guide** | 日常的な開発（バグ修正、小規模な追加）で効率を最大化する手法。 |
| **The Longform Guide** | 大規模なリファクタリングや、新規プロジェクトの立ち上げ時の設計指針。 |
| **The Security Guide** | 安全なコードを書くための、ECCのセキュリティ機能のフル活用法。 |

---

## 🗺 アーキテクチャ文書 (`docs/`)
システムの裏側を知るための専門文書です。

- **`COMMAND-AGENT-MAP.md`**: コマンドとエージェントの紐付け一覧（今回のインデックスの基礎となった資料）。
- **`TECH-STACK.md`**: ECCが自律的に選択する技術スタックの選定基準。
- **`AGENT-ORCHESTRATION.md`**: 複数のエージェントをどのように連携させるかの設計思想。
- **`COMPARISON.md`**: 他のAIコーディングツールとECCの機能・思想の比較。

---

## 🌍 多言語対応 (Multilingual)
ECCはグローバルに展開されており、`docs/` 以下に多言語版が用意されています。

- **`ja-JP/`**: 日本語版ドキュメント。主要なガイドやルールが翻訳されています。
- **その他の言語**: `zh-CN`（簡体字）, `zh-TW`（繁体字）, `ko-KR`（韓国語）, `pt-BR`（ポルトガル語）, `tr`（トルコ語）。

---

## 📝 テンプレートと例 (`examples/`)
新しいプロジェクトを始める際の「見本」です。

- **`.claude.example`**: `CLAUDE.md` のスターターテンプレート。
- **`.cursorrules.example`**: Cursor用の設定例。
- **`workflows/`**: GitHub ActionsなどのCI設定のサンプル。

---

## 🛠 学習の進め方（おすすめの順番）

1.  まず、**`README.md`** で全体を俯瞰する。
2.  **`The Shortform Guide`** で、毎日の開発における `/plan` や `/tdd` の使い方を身につける。
3.  詰まったら **`COMMANDS-QUICK-REF.md`**（コマンド早見表）を見る。
4.  アーキテクチャに興味が出たら **`docs/`** 以下の専門文書を深掘りする。

---

**依存関係**:
- **同期**: `doc-updater` エージェントが、コードの変更に合わせてこれらのドキュメントを最新に保つ役割を担っています。
- **活用**: エージェント自身も `docs-lookup` スキルを通じて、これらの文書を「自分のマニュアル」として参照します。
