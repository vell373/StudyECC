# Security Bounty Hunter スキル

バグバウンティプログラムのスコープ内でセキュリティ脆弱性を体系的に探索する。

## スコープ内のパターン（高価値ターゲット）

### SSRF（サーバーサイドリクエストフォージェリ）
```
確認箇所:
- URLパラメータを受け取るエンドポイント
- Webhookの設定
- 画像・ファイルのインポート機能
- メタデータエンドポイント (169.254.169.254)
```

### 認証バイパス
```
確認箇所:
- JWT検証の欠如（alg: none攻撃）
- パスワードリセットフローの欠陥
- OAuth状態パラメータの欠如
- セッション固定攻撃
- マルチテナント環境での水平権限昇格
```

### RCE（リモートコード実行）
```
確認箇所:
- テンプレートインジェクション (SSTI)
- デシリアライゼーション
- ファイルアップロード後の実行
- コマンドインジェクション
- eval()やsimilar関数の悪用
```

### SQLインジェクション
```
確認箇所:
- ユーザー入力を使ったクエリ
- ORDER BY句
- LIKE句のワイルドカード
- ストアドプロシージャ
- バッチクエリ
```

### コマンドインジェクション
```
確認箇所:
- system(), exec(), shell_exec()
- ファイル名処理
- zipやtarの引数
- FFmpegやImageMagickのパラメータ
```

### パストラバーサル
```
確認箇所:
- ファイルダウンロードエンドポイント
- ファイル読み込みAPI
- zipの展開
- テンプレートのインクルード
```

### XSS（クロスサイトスクリプティング）
```
確認箇所:
- ユーザー入力の反射
- DOM操作 (innerHTML)
- CSPの設定不備
- HTMLサニタイズのバイパス
```

## スキップするパターン（スコープ外）

- 自分のアカウントへの攻撃のみ
- レート制限（重大な影響がない場合）
- Self-XSS
- CSRFでの低リスク操作（ログアウト等）
- SSL/TLSの設定問題（PCI-DSSスコープ外）
- 公開情報の列挙

## 7ステップのワークフロー

### ステップ1: スコープの確認
```
確認事項:
- 対象ドメイン・IPレンジ
- 除外対象
- 支払いポリシー
- 責任ある開示ポリシー
```

### ステップ2: 偵察
```bash
# サブドメイン列挙
subfinder -d target.com
amass enum -d target.com

# エンドポイント探索
waybackurls target.com
gau target.com

# JS解析
linkfinder -i https://target.com -o cli
```

### ステップ3: 攻撃面のマッピング
```
識別対象:
- 認証エンドポイント
- ファイルアップロード機能
- APIエンドポイント
- 外部サービスの統合
- 管理画面
```

### ステップ4: 自動スキャン
```bash
# 基本的な脆弱性スキャン
nuclei -u https://target.com -t cves/
sqlmap -u "https://target.com/api?id=1" --batch
```

### ステップ5: 手動テスト

高価値パターンに集中する:
```python
# SSRF テストの例
payloads = [
    "http://169.254.169.254/latest/meta-data/",
    "http://localhost:6379/",  # Redis
    "file:///etc/passwd",
    "http://internal-api.company.com/",
]

# JWT alg:none テスト
header = base64.b64encode(b'{"alg":"none","typ":"JWT"}')
# 署名なしでのアクセス試行
```

### ステップ6: 影響の証明
- Proof of Concept (PoC) を作成する
- 実際の影響（データ流出、アカウント乗っ取り等）を示す
- スクリーンショットとリクエスト/レスポンスのログを取る

### ステップ7: レポート作成

## レポートテンプレート

```markdown
## タイトル
[脆弱性タイプ] in [機能/エンドポイント] が [影響] につながる

## 深刻度
Critical / High / Medium / Low

## CVSS スコア
[スコア] ([ベクター文字列])

## 概要
[2-3文で脆弱性を説明]

## 再現手順
1. [ステップ1]
2. [ステップ2]
3. [スクリーンショットまたはリクエスト/レスポンス]

## 影響
[攻撃者が何を達成できるか]

## 修正案
[推奨する修正方法]

## 参考資料
- [OWASP リンク]
- [CVE リンク（該当する場合）]
```

## 倫理的なガイドライン

- スコープ外のシステムには触れない
- 脆弱性を悪用してデータを取得しない
- 責任ある開示のタイムラインを守る
- 実際の顧客データを使ったテストを行わない
- 発見した脆弱性の情報を第三者と共有しない
