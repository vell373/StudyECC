# Web デザイン品質基準

## テンプレート対策

汎用テンプレートのような UI を出荷しない。フロントエンドアウトプットは意図的、独断的、製品固有に見えるべき。

### 禁止パターン

- 均一なスペーシングと階層なしのデフォルトカードグリッド
- 中央配置されたヘッドライン、グラデーションの blob、汎用 CTA を持つストックヒーロセクション
- 変更されていないライブラリデフォルトが完成デザインとして渡されること
- レイヤリング、深度、またはモーションなし平坦レイアウト
- すべてのコンポーネント全体で均一な radius、spacing、shadow
- 1 つの装飾的なアクセントカラーを持つセーフグレイオンホワイトスタイリング
- サイドバー + カード + チャート とポイント オブ ビュー なし ダッシュボード-バイ-ナンバーズレイアウト
- 慎重な理由のなくデフォルトフォントスタック使用

### 必須の品質

すべての意味のあるフロントエンド表面は、少なくとも 4 つを示すべき:

1. スケール対比による明確な階層
2. 均一なパディング全体ではなく、スペーシングに意図的なリズム
3. 重複、影、表面、またはモーションによる深度またはレイヤリング
4. 文字を持つタイポグラフィと実際のペアリング戦略
5. 装飾的ではなく意味的に使用される色
6. ホバー、フォーカス、アクティブ状態が設計される感覚
7. 適切なグリッド破壊編集または bento コンポジション
8. 視覚方向に合わせたテクスチャ、粒度、または雰囲気
9. フロー を明確にする代わりにモーションはそれからそらします
10. データ可視化はアフターソートではなく設計システムの一部として扱われる

## フロントエンドコード書き込み前

1. 特定のスタイル方向を選択。「クリーン最小」のような曖昧なデフォルト避け。
2. パレットを意図的に定義。
3. タイポグラフィを意図的に選択。
4. 少なくとも小さいセットの実際の参照を集める。
5. ECC デザイン/フロントエンドスキルを関連する場所で使用。

## 価値のあるスタイル方向

- 編集/雑誌
- Neo-brutalism
- 実際の深度を持つ Glassmorphism
- ダーク豪華または軽い豪華な規律の対比
- Bento レイアウト
- Scrollytelling
- 3D 統合
- スイス/国際
- レトロ未来主義

デフォルトでダークモードに自動的に既定しないこと。製品が実際に望む視覚方向選択。
