# PHP セキュリティ

> このファイルは [common/security.md](../common/security.md) を PHP 固有のコンテンツで拡張します。

## 入力と出力

- フレームワーク境界でリクエスト入力を検証（`FormRequest`、Symfony Validator、または明示的 DTO 検証）。
- テンプレートで出力をエスケープ；生 HTML レンダリングを例外として正当化が必須。
- クエリパラメータ、Cookie、ヘッダー、アップロードされたファイルメタデータを検証なしで信頼しない。

## データベースセーフティ

- すべての動的クエリに準備ステートメント（`PDO`、Doctrine、Eloquent クエリビルダー）を使用。
- コントローラー/ビューで SQL 文字列ビルディングを避ける。
- ORM マスアサインを慎重にスコープし、書き込み可能フィールドをホワイトリスト化。

## シークレットと依存関係

- コミット設定ファイルではなく、環境変数またはシークレットマネージャーからシークレットをロード。
- CI で `composer audit` を実行し、依存関係を追加する前に新規パッケージメンテナー信頼を確認。
- 意図的にメジャーバージョンをピン留めし、放棄されたパッケージをすぐに削除。

## 認証とセッションセーフティ

- パスワード保存に `password_hash()` / `password_verify()` を使用。
- 認証と権限変更後にセッション識別子を再生成。
- 状態変更 web リクエストで CSRF 保護を強制。

## リファレンス

スキル `laravel-security` で Laravel 固有セキュリティガイダンスを参照。
