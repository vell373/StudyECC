# PHP コーディングスタイル

> このファイルは [common/coding-style.md](../common/coding-style.md) を PHP 固有のコンテンツで拡張します。

## 標準

- **PSR-12** フォーマットと命名規約に従う。
- アプリケーションコードに `declare(strict_types=1);` を優先。
- スカラー型ヒント、戻り値型、型指定プロパティをすべての新規コードで使用。

## イミュータビリティ

- サービス境界を越えるデータ用に不変 DTO と値オブジェクトを優先。
- リクエスト/レスポンスペイロードに `readonly` プロパティまたは不変コンストラクタを可能な限り使用。
- シンプルなマップにはアレイを保持；ビジネスクリティカル構造は明示的クラスに昇格。

## フォーマット

- フォーマット用に **PHP-CS-Fixer** または **Laravel Pint** を使用。
- 静的解析に **PHPStan** または **Psalm** を使用。
- Composer スクリプトをチェックイン済みにして、ローカルと CI で同じコマンドが実行される。

## インポート

- すべての参照クラス、インターフェース、トレイトに `use` ステートメントを追加。
- プロジェクト明示的に完全修飾名を優先しない限り、グローバル名前空間に依存しない。

## エラーハンドリング

- 例外状態には例外をスロー；新規コードで隠れたエラーチャネルとして `false`/`null` を返さない。
- フレームワーク/リクエスト入力をドメインロジックに達する前に検証 DTO に変換。

## リファレンス

スキル `backend-patterns` でサービス/リポジトリレイヤリングガイダンスを参照。
