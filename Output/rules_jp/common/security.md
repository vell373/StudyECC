# セキュリティガイドライン

## 必須セキュリティチェック

ANY コミット前に:
- [ ] ハードコードされたシークレット（APIキー、パスワード、トークン）がない
- [ ] すべてのユーザー入力が検証されている
- [ ] SQL インジェクション防止（パラメータ化されたクエリ）
- [ ] XSS 防止（サニタイズされたHTML）
- [ ] CSRF 保護が有効になっている
- [ ] 認証/認可が検証されている
- [ ] すべてのエンドポイントでレート制限が有効
- [ ] エラーメッセージが機密データを漏らしていない

## シークレット管理

- ソースコードにシークレットをハードコーディングするな（決定事項）
- 常に環境変数またはシークレットマネージャーを使用する（必須）
- 起動時に必須シークレットが存在することを検証する
- 露出した可能性のあるシークレットをローテーションする

## セキュリティ対応プロトコル

セキュリティ問題が見つかった場合:
1. 直ちにSTOP
2. **security-reviewer** エージェントを使用
3. 続行する前にCRITICAL問題を修正
4. 露出したシークレットをローテーション
5. 類似問題についてコード全体をレビュー
