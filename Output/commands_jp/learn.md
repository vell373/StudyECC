# /learn - 再利用可能なパターンを抽出

現在のセッションを解析して、スキルとして保存する価値のあるパターンを抽出します。

## トリガー

セッション中、非自明な問題を解決したときに `/learn` を実行します。

## 抽出する内容

以下を探します：

1. **エラー解決パターン**
   - どのエラーが発生しましたか?
   - 根本原因は何でしたか?
   - 何が修正しましたか?
   - 同様のエラーで再利用可能ですか?

2. **デバッグテクニック**
   - 非自明なデバッグステップ
   - 動作したツール組み合わせ
   - 診断パターン

3. **回避策**
   - ライブラリの癖
   - API制限
   - バージョン固有の修正

4. **プロジェクト固有のパターン**
   - 発見されたコードベース規約
   - 行われたアーキテクチャ決定
   - 統合パターン

## 出力形式

`~/.claude/skills/learned/[pattern-name].md` にスキルファイルを作成：

```markdown
# [記述的パターン名]

**抽出日:** [日付]
**コンテキスト:** [このパターンが適用される時についての簡単な説明]

## 問題
[この問題を解決するもの - 具体的に]

## 解決策
[パターン/テクニック/回避策]

## 例
[該当する場合はコード例]

## いつ使うか
[トリガー条件 - このスキルを活動させるべき]
```

## プロセス

1. セッションから抽出可能なパターンを確認
2. 最も価値がある/再利用可能なインサイトを特定
3. スキルファイルをドラフト
4. 保存前にユーザーに確認を質問
5. `~/.claude/skills/learned/` に保存

## ノート

- 自明な修正（タイプミス、単純な構文エラー）を抽出しない
- 一度限りの問題（特定のAPI停止など）を抽出しない
- 将来のセッションで時間を節約するパターンに焦点
- スキルを焦点を絞ったものに保つ - 1スキルあたり1パターン
