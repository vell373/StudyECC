# /learn - 再利用可能なパターンを抽出

現在のセッションを解析し、スキルとして保存する価値のあるパターンを抽出します。

## トリガー

セッション中に非自明な問題を解決したときに任意の時点で `/learn` を実行します。

## 抽出対象

以下を探します:

1. **エラー解決パターン**
   - どのエラーが発生したか?
   - 根本原因は何か?
   - それを修正したのは何か?
   - これは同様のエラーで再利用可能か?

2. **デバッグ技術**
   - 明白ではないデバッグステップ
   - 機能したツール組み合わせ
   - 診断パターン

3. **回避策**
   - ライブラリの癖
   - API 制限
   - バージョン固有の修正

4. **プロジェクト固有パターン**
   - 発見されたコードベース規約
   - 下されたアーキテクチャ決定
   - 統合パターン

## 出力フォーマット

`~/.claude/skills/learned/[pattern-name].md` にスキルファイルを作成:

```markdown
# [説明的なパターン名]

**抽出:** [日付]
**コンテキスト:** [これが適用される場合の簡単な説明]

## 問題
[このパターンが解決する問題 - 具体的に]

## 解決策
[パターン/技術/回避策]

## 例
[該当する場合はコード例]

## 使用時機
[トリガー条件 - これをアクティベートすべき理由]
```

## プロセス

1. セッションを抽出可能なパターンについてレビュー
2. 最も価値のある/再利用可能な洞察を特定
3. スキルファイルの下書き
4. 保存前にユーザーに確認を求める
5. `~/.claude/skills/learned/` に保存

## 注記

- 些細な修正（タイプミス、単純な構文エラー）を抽出しない
- 一時的な問題（特定の API 停止など）を抽出しない
- 将来のセッションで時間を節約するパターンに焦点
- スキルを焦点を絞る - スキルごとに 1 つのパターン
