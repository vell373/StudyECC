# ループ状態コマンド

アクティブなループの状態、進捗、失敗シグナルを検査します。

このスラッシュコマンドは現在のセッションが队列から削除した後にのみ実行できます。
くさび止め状態のセッションまたは兄弟セッションを検査する必要がある場合は、別のターミナルからパッケージ化されたCLIを実行してください：

```bash
npx --package ecc-universal ecc loop-status --json
```

CLIは `~/.claude/projects/**` の下のローカルClaudeトランスクリプトJSONLファイルをスキャンし、
古い `ScheduleWakeup` 呼び出しまたは対応する `tool_result` がない `Bash` ツール呼び出しを報告します。

## 使用方法

`/loop-status [--watch]`

## 報告する内容

- アクティブなループパターン
- 現在のフェーズと最後に成功したチェックポイント
- 失敗チェック（ある場合）
- 推定時間/コストのドリフト
- 推奨される介入（続行/一時停止/停止）

## クロスセッションCLI

- `ecc loop-status --json` 最近のローカルClaudeトランスクリプト用のマシン読み取り可能な状態を出力します。
- `ecc loop-status --home <dir>` 別のホームディレクトリをスキャンします（別のローカルプロファイルまたはマウントされたワークスペースを検査する場合）。
- `ecc loop-status --transcript <session.jsonl>` 1つのトランスクリプトを直接検査します。
- `ecc loop-status --bash-timeout-seconds 1800` 古いBashしきい値を調整します。
- `ecc loop-status --exit-code` 古いループまたはツール信号が見つかった場合は `2` で、トランスクリプトをスキャンできない場合は `1` で終了します。
- `--exit-code` と `--watch` を使用する場合は `--watch-count` が必要です（ウォッチドッグスクリプトがプロセス終了を永遠に待たないようにするため）。
- `ecc loop-status --watch` は割り込まれるまで状態を定期的に更新します。
- `ecc loop-status --watch --watch-count 3 --exit-code` 限定された回数更新してから、見られた最高ステータスで終了します。
- `ecc loop-status --watch --watch-count 3` スクリプトとハンドオフ用の限定されたウォッチストリームを発行します。
- `ecc loop-status --watch --write-dir ~/.claude/loops` `index.json` とセッションごとのJSONスナップショットを保守します（兄弟ターミナルまたはウォッチドッグスクリプト用）。

## ウォッチモード

`--watch` が存在する場合、定期的に状態を更新します。`--json` を使用すると、各更新が1行あたり1つのJSONオブジェクトとして発行されるため、別のターミナルまたはスクリプトがストリームを消費できます。

## スナップショットファイル

現在のClaudeセッションが `ループ状態` をキューイングするのを待つことなく、別のプロセスがループ状態を検査する必要がある場合は `--write-dir <dir>` を使用してください。CLIは以下を書き込みます：

- 検査されたセッションごとに1行の `index.json`。
- そのセッションの完全なステータスペイロード `<session-id>.json`。

これらのファイルはローカルトランスクリプト分析のスナップショットです。Claude Codeランタイムツール呼び出しを制御またはタイムアウトさせません。

## 引数

$ARGUMENTS:
- `--watch` オプション
