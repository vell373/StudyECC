# PRPプラン実行コマンド

このコマンドは実装プランを実行し、複数の検証レベルを適用します。

デフォルトではインラインで実行します。Taskツールまたはサブエージェントをデフォルトでは呼び出しません。これにより、エージェントファイルがないプラグインインストールから提供されるコマンドでも `/prp-implement` が使用可能になります。

## このコマンドの機能

1. **実装プランを検出** — 進行中の計画や承認済みプランを特定
2. **段階的に実行** — フェーズを順序立てて実行
3. **複数レベルで検証** — 静的解析、ユニットテスト、ビルド、統合テスト、エッジケース
4. **検証失敗で停止** — リグレッションを防ぐため、テスト失敗時に次フェーズへの進行を阻止

## 使用時

以下の場合に `/prp-implement` を使用してください:
- `/prp-plan` で作成した承認済みプランを実行する
- 複数のコンポーネント変更を厳密に検証したい
- リグレッションのリスクが高い場合
- 本番環境へのデプロイ前に品質を保証したい

## 動作方法

アシスタントは以下を実行します:

### フェーズ1: 検出（DETECT）
- 現在のプロジェクトを特定
- 進行中の `/prp-plan` プランまたは承認済みプランを検索
- プランが見つからない場合は停止: 「有効なプランが見つかりません。`/prp-plan` でプランを作成してください。」

### フェーズ2: 読み込み（LOAD）
- プランのファイルを読み込む
- 各フェーズの説明、依存関係、実装対象を抽出
- フェーズの順序を検証し、ユーザーに表示

### フェーズ3: 準備（PREPARE）
- 現在のコードベースをスキャン
- 実装対象ファイルが存在するか、変更可能か確認
- 依存関係の競合がないか検査
- 準備状況をユーザーに報告

### フェーズ4: 実行（EXECUTE）
各フェーズについて:
1. フェーズの説明と実装内容を表示
2. コード生成・編集を実行
3. ユーザー確認を待つ（可能な場合）

### フェーズ5: 検証（VALIDATE）

**検証レベル1 — 静的解析**
- 構文チェック（TypeScript: `tsc --noEmit`、Python: `mypy`、Go: `go vet`）
- リント実行（ESLint, Pylint, golangci-lint）
- 型安全性確認

**検証レベル2 — ユニットテスト**
- テストターゲット自動検出（`*.test.*`, `*.spec.*`, `test_*.py`）
- テスト実行（Jest, vitest, pytest）
- カバレッジ確認（ターゲット: 80%+）
- テスト失敗時は停止: 「テストが失敗しました。コードを修正してから再実行してください。」

**検証レベル3 — ビルドチェック**
- 言語別ビルドコマンド実行（`npm run build`, `python setup.py build`, `go build`）
- ビルドエラー時は停止

**検証レベル4 — 統合テスト**
- エンドツーエンドテスト実行（Playwright, pytest 統合テスト）
- 外部統合（API、DB）をテスト
- 失敗時は停止

**検証レベル5 — エッジケーステスト**
- 入力値の境界ケースをテスト
- エラーハンドリング検証
- パフォーマンス基準確認

### フェーズ6: 報告（REPORT）

すべての検証が通った場合、ユーザーに報告:

```
## 実装完了

**実装フェーズ:**
- フェーズ1: 〇 (3ファイル変更)
- フェーズ2: 〇 (2ファイル追加)
- フェーズ3: 〇 (1ファイル更新)

**検証結果:**
- 静的解析: ✓ pass
- ユニットテスト: ✓ 42/42 pass (カバレッジ: 87%)
- ビルド: ✓ success
- 統合テスト: ✓ pass
- エッジケース: ✓ pass

**推奨次ステップ:**
- `git push` → リモートへプッシュ
- `/prp-pr` → プルリクエストを作成
- `/code-review` → 追加レビュー実施
```

## 使用例

```
ユーザー: /prp-implement

アシスタント: (プランを検出)
プランが見つかりました: 「リアルタイムマーケット解決通知」

実装フェーズ:
- フェーズ1: データベーススキーマ
- フェーズ2: 通知サービス
- フェーズ3: 統合ポイント
- フェーズ4: フロントエンドコンポーネント

準備状況: ✓ すべてのターゲットファイルが変更可能

フェーズ1を開始しますか? (はい/いいえ/確認)
```

## 重要な注意事項

**テスト失敗で必ず停止**: 検証レベル2以上でテストが失敗した場合、自動で次のフェーズへ進みません。ユーザーがコードを修正してから再度実行する必要があります。

**段階的な確認**: 各フェーズ完了後、ユーザー確認を待ちます（--auto-confirm フラグで自動化可能）。

## 他のコマンドとの統合

実装前:
- `/prp-plan` で包括的プランを作成

実装後:
- `/prp-pr` → プルリクエストを作成
- `/code-review` → コードレビューを実施
