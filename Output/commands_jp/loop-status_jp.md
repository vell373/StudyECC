# ループステータスコマンド

アクティブなループの状態、進捗、失敗シグナルを検査します。

このスラッシュコマンドは、現在のセッションがそれをデキュー後にのみ実行できます。ウェッジされたまたはシブリングセッションを検査する必要がある場合は、別のターミナルからパッケージされた CLI を実行します:

```bash
npx --package ecc-universal ecc loop-status --json
```

CLI は `~/.claude/projects/**` 下のローカル Claude トランスクリプト JSONL ファイルをスキャンし、古い `ScheduleWakeup` コールまたは対応する `tool_result` がないスタンド `Bash` ツールコールを報告します。

## 使用方法

`/loop-status [--watch]`

## 何を報告するか

- アクティブなループパターン
- 現在のフェーズと最後に成功したチェックポイント
- 失敗したチェック（ある場合）
- 推定時間/コストドリフト
- 推奨される介入（続行/一時停止/停止）

## クロスセッション CLI

- `ecc loop-status --json` は最近のローカル Claude トランスクリプト用のマシンが読める状態を出力します。
- `ecc loop-status --home <dir>` は別のローカルプロフィールまたはマウントされたワークスペースを検査する場合に別のホームディレクトリをスキャン。
- `ecc loop-status --transcript <session.jsonl>` は 1 つのトランスクリプトを直接検査。
- `ecc loop-status --bash-timeout-seconds 1800` は古い Bash しきい値を調整。
- `ecc loop-status --exit-code` は古いループまたはツールシグナルが見つかった場合は `2` で終了、またはトランスクリプトがスキャンできない場合は `1`。
- `--exit-code` + `--watch` には `--watch-count` が必要です。ウォッチドッグスクリプトが永遠にプロセス終了を待たないようにするため。
- `ecc loop-status --watch` は割り込まれるまでステータスをリフレッシュ。
- `ecc loop-status --watch --watch-count 3 --exit-code` は制限された回数をリフレッシュしてから、見られた最高ステータスで終了。
- `ecc loop-status --watch --watch-count 3` はスクリプトとハンドオフ用に制限されたウォッチストリームを出力。
- `ecc loop-status --watch --write-dir ~/.claude/loops` は `index.json` とセッションごと JSON スナップショットを保守（シブリング端末またはウォッチドッグスクリプト用）。

## ウォッチモード

`--watch` が存在する場合、定期的にステータスをリフレッシュします。`--json` の場合、各リフレッシュは 1 行ごとに 1 つの JSON オブジェクトとして出力され、別の端末またはスクリプトはストリームを消費できます。

## スナップショットファイル

別のプロセスが現在の Claude セッションがデキュー待たずにループ状態を検査する必要がある場合、`--write-dir <dir>` を使用します。CLI は以下を書き込みます:

- 検査されたセッションごと 1 行を持つ `index.json`。
- そのセッションの完全なステータスペイロード付き `<session-id>.json`。

これらのファイルはローカルトランスクリプト解析のスナップショット。これらは Claude Code ランタイムツールコールを制御または タイムアウトしません。

## 引数

$ARGUMENTS:
- `--watch`（オプション）
