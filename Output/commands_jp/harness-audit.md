# ハーネス監査コマンド

決定論的なリポジトリハーネス監査を実行し、優先度付きスコアカードを返します。

## 使用方法

`/harness-audit [scope] [--format text|json] [--root path]`

- `scope`（オプション）: `repo`（デフォルト）、`hooks`、`skills`、`commands`、`agents`
- `--format`: 出力スタイル（デフォルト `text`、自動化用は `json`）
- `--root`: カレントワーキングディレクトリの代わりに特定のパスを監査

## 決定論的エンジン

常に実行：

```bash
node scripts/harness-audit.js <scope> --format <text|json> [--root <path>]
```

このスクリプトは、スコアリングとチェックの正式なソースです。追加の次元やアドホックなポイントを発明しないでください。

Rubricバージョン: `2026-03-30`。

スクリプトは7つの固定カテゴリ（各0-10正規化）を計算：

1. ツールカバレッジ
2. コンテキスト効率
3. 品質ゲート
4. メモリ永続化
5. 評価カバレッジ
6. セキュリティガードレール
7. コスト効率

スコアは明示的なファイル/ルールチェックから導出され、同じコミットの場合は再現可能です。スクリプトはデフォルトでカレントワーキングディレクトリを監査し、対象がECCリポジトリ自体か、ECCを使用するコンシューマープロジェクトかを自動検出します。

## 出力契約

以下を返します：

1. `overall_score` / `max_score`（`repo` の場合は70; スコープが限定される場合はより小さい）
2. カテゴリスコアと具体的な調査結果
3. 失敗したチェック（正確なファイルパス付き）
4. 決定論的出力からのトップ3アクション（`top_actions`）
5. 次に適用するECCスキルを提案

## チェックリスト

- スクリプト出力を直接使用; 手動でスコア再計算しない
- `--format json` がリクエストされた場合、スクリプトJSONを変更せずに返す
- テキストがリクエストされた場合、失敗したチェックとトップアクションをサマリー
- `checks[]` と `top_actions[]` の正確なファイルパスを含める

## 結果の例

```text
ハーネス監査（repo）: 66/70
- ツールカバレッジ: 10/10（10/10 pts）
- コンテキスト効率: 9/10（9/10 pts）
- 品質ゲート: 10/10（10/10 pts）

トップ3アクション：
1) [セキュリティガードレール] hooks/hooks.json にプロンプト/ツールプリフライトセキュリティガードを追加します。（hooks/hooks.json）
2) [ツールカバレッジ] commands/harness-audit.md と .opencode/commands/harness-audit.md を同期します。（.opencode/commands/harness-audit.md）
3) [評価カバレッジ] scripts/hooks/lib 全体の自動テストカバレッジを増加させます。（tests/）
```

## 引数

$ARGUMENTS:
- `repo|hooks|skills|commands|agents`（オプションスコープ）
- `--format text|json`（オプション出力形式）
