# Evolve コマンド

## 実装

プラグインルートパスを使用して instinct CLI を実行：

```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/continuous-learning-v2/scripts/instinct-cli.py" evolve [--generate]
```

または `CLAUDE_PLUGIN_ROOT` が設定されていない場合（手動インストール）：

```bash
python3 ~/.claude/skills/continuous-learning-v2/scripts/instinct-cli.py evolve [--generate]
```

インスティンクトを解析し、関連するインスティンクトをより高レベルの構造にクラスタリング：
- **コマンド**: ユーザーが呼び出す操作を説明するインスティンクト
- **スキル**: 自動トリガーされる動作を説明するインスティンクト
- **エージェント**: 複雑で複数のステップのプロセスを説明するインスティンクト

## 使用方法

```
/evolve                    # すべてのインスティンクトを解析し、進化を提案
/evolve --generate         # さらに evolved/{skills,commands,agents} 下のファイルを生成
```

## 進化ルール

### コマンド（ユーザーが呼び出し）
インスティンクトがユーザーが明示的に要求するアクションを説明する場合：
- ユーザーが「...するよう依頼した場合」について複数のインスティンクト
- 「新しい X を作成する場合」のようなトリガーを持つインスティンクト
- 反復可能なシーケンスに従うインスティンクト

例：
- `new-table-step1`: 「データベーステーブルを追加する場合、マイグレーションを作成」
- `new-table-step2`: 「データベーステーブルを追加する場合、スキーマを更新」
- `new-table-step3`: 「データベーステーブルを追加する場合、型を再生成」

→ 作成: **new-table** コマンド

### スキル（自動トリガー）
インスティンクトが自動的に発生すべき動作を説明する場合：
- パターンマッチングトリガー
- エラーハンドリング応答
- コードスタイル実装

例：
- `prefer-functional`: 「関数を書く場合、関数型スタイルを優先」
- `use-immutable`: 「状態を修正する場合、イミュータブルパターンを使用」
- `avoid-classes`: 「モジュールを設計する場合、クラスベース設計を回避」

→ 作成: `functional-patterns` スキル

### エージェント（深さ/分離が必要）
インスティンクトが分離からメリットを得られる複雑で複数のステップのプロセスを説明する場合：
- デバッグワークフロー
- リファクタリングシーケンス
- リサーチタスク

例：
- `debug-step1`: 「デバッグする場合、まずログを確認」
- `debug-step2`: 「デバッグする場合、失敗したコンポーネントを分離」
- `debug-step3`: 「デバッグする場合、最小限の再現を作成」
- `debug-step4`: 「デバッグする場合、テストで修正を確認」

→ 作成: **debugger** エージェント

## 何をするか

1. 現在のプロジェクトコンテキストを検出
2. プロジェクト + グローバルインスティンクトを読み込み（ID 競合時はプロジェクトが優先）
3. インスティンクトをトリガー/ドメインパターンでグループ化
4. 以下を特定：
   - スキル候補（2 つ以上のインスティンクトでトリガークラスタリング）
   - コマンド候補（高信頼度ワークフローインスティンクト）
   - エージェント候補（より大きく、高信頼度のクラスタ）
5. 昇進候補を表示（プロジェクト → グローバル）（該当する場合）
6. `--generate` が渡された場合、ファイルを以下に書き込み：
   - プロジェクトスコープ: `~/.claude/homunculus/projects/<project-id>/evolved/`
   - グローバルフォールバック: `~/.claude/homunculus/evolved/`

## フラグ

- `--generate`: 解析出力に加えて、進化したファイルを生成

## 生成されたファイル形式

### コマンド
```markdown
---
name: new-table
description: マイグレーション、スキーマ更新、型生成を含む新しいデータベーステーブルを作成
command: /new-table
evolved_from:
  - new-table-migration
  - update-schema
  - regenerate-types
---

# 新しいテーブルコマンド

[クラスタ化されたインスティンクトに基づいて生成されたコンテンツ]

## ステップ
1. ...
2. ...
```

### スキル
```markdown
---
name: functional-patterns
description: 関数型プログラミングパターンを実装
evolved_from:
  - prefer-functional
  - use-immutable
  - avoid-classes
---

# 関数型パターンスキル

[クラスタ化されたインスティンクトに基づいて生成されたコンテンツ]
```

### エージェント
```markdown
---
name: debugger
description: 体系的なデバッグエージェント
model: sonnet
evolved_from:
  - debug-check-logs
  - debug-isolate
  - debug-reproduce
---

# デバッガーエージェント

[クラスタ化されたインスティンクトに基づいて生成されたコンテンツ]
```
