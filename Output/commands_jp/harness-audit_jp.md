# ハーネス監査コマンド

リポジトリハーネスの決定論的監査を実行し、優先順位付きスコアカードを返します。

## 使用方法

`/harness-audit [scope] [--format text|json] [--root path]`

- `scope`（オプション）: `repo`（デフォルト）、`hooks`、`skills`、`commands`、`agents`
- `--format`: 出力スタイル（デフォルト `text`、自動化向け `json`）
- `--root`: 現在の作業ディレクトリの代わりに特定のパスを監査

## 決定論的エンジン

常に以下を実行:

```bash
node scripts/harness-audit.js <scope> --format <text|json> [--root <path>]
```

このスクリプトはスコアリングとチェックの情報源です。追加の次元またはアドホックなポイントを自分で発明しないでください。

ルールブックバージョン: `2026-03-30`。

スクリプトは 7 つの固定カテゴリー（各 `0-10` に正規化）を計算します:

1. ツールカバレッジ
2. コンテキスト効率
3. 品質ゲート
4. メモリ永続性
5. Eval カバレッジ
6. セキュリティガードレール
7. コスト効率

スコアは明示的なファイル/ルールチェックから導出され、同じコミットに対して再現可能です。
スクリプトはデフォルトで現在の作業ディレクトリを監査し、ターゲットが ECC リポジトリ自体か、ECC を使用するコンシューマープロジェクトかを自動検出します。

## 出力契約

返す:

1. `overall_score` （`max_score` のうち）（`repo` の場合 70、スコープ限定の場合はより小さい）
2. カテゴリスコアと具体的な調査結果
3. 失敗したチェック（正確なファイルパス付き）
4. 決定論的出力からの上位 3 つのアクション（`top_actions`）
5. 次に適用する推奨 ECC スキル

## チェックリスト

- スクリプト出力を直接使用。手動で再採点しないでください。
- `--format json` が要求されている場合、スクリプト JSON を変更なしで返します。
- テキストが要求されている場合、失敗したチェックと上位アクションを要約します。
- `checks[]` と `top_actions[]` から正確なファイルパスを含めます。

## 結果例

```text
ハーネス監査（repo）: 66/70
- ツールカバレッジ: 10/10 (10/10 pts)
- コンテキスト効率: 9/10 (9/10 pts)
- 品質ゲート: 10/10 (10/10 pts)

上位 3 つのアクション:
1) [セキュリティガードレール] hooks/hooks.json にプロンプト/ツール事前ガードを追加。(hooks/hooks.json)
2) [ツールカバレッジ] commands/harness-audit.md と .opencode/commands/harness-audit.md を同期。(.opencode/commands/harness-audit.md)
3) [Eval カバレッジ] scripts/hooks/lib 全体の自動テストカバレッジを増加。(tests/)
```

## 引数

$ARGUMENTS:
- `repo|hooks|skills|commands|agents`（オプションスコープ）
- `--format text|json`（オプション出力形式）
