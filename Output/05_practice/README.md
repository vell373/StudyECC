# 05_practice — 写経・再実装の作業場

ECCのコードを読んで理解したものを、実際に手を動かして確認する場所。
「分かったつもり」を防ぎ、本当の理解を定着させる。

## ディレクトリ構成

```
05_practice/
├── README.md           （このファイル）
├── agents/             （エージェントの再実装練習）
├── skills/             （スキルの再実装練習）
├── hooks/              （フックの再実装練習）
└── mini-harness/       （最小構成のハーネスを自力で作る）
```

## 使い方

1. ECC の対象ファイルを読む
2. ファイルを閉じて、自分でゼロから書いてみる
3. 書いた後に ECC の元ファイルと比較する
4. 差分・気づきを 02_notes に記録する

## ファイル命名規則

```
{ECC対応ファイル名}_practice.md

例:
  fb-skill_practice.md       → ECC/skills/fb.md の再実装
  router-agent_practice.md   → ルーティングエージェントの自作
```
