# EverythingClaudeCode 学習プロジェクト

このプロジェクトは、**ECC（EverythingClaudeCode）** の高度なアーキテクチャを静的解析し、「**ハーネスエンジニアリング**」や「**コンテキストエンジニアリング**」のスキルを習得・実践するための学習リポジトリです。

> **元リポジトリ (Reference)**: [affaan-m/everything-claude-code](https://github.com/affaan-m/everything-claude-code)

ECCの巨大な知識ベース（エージェント、スキル、ルール、フック）を解剖し、最終的には「ECCを参照せずに、同等レベルのAI開発ハーネスを自力で設計・実装できるようになること」を目標としています。

---

## 📂 プロジェクト構造

プロジェクトは大きく「読み取り専用の解析対象（ECC）」と「書き込み可能な学習成果物（Output）」に分かれています。

```text
StudyECC/
├── ECC/        ← 🔍 解析対象（変更不可・巨大なナレッジベース）【Git管理対象外】
├── Output/     ← ✍️ 学習アウトプット（このリポジトリのメイン成果物）
├── CLAUDE.md   ← 🤖 AIアシスタント用の振る舞い・ルール定義
└── README.md   ← 📖 このファイル
```

※ `ECC/` ディレクトリ配下はGitの管理対象外に設定されており、純粋な参照用データとして扱います。

## ✍️ `Output/` フォルダの構成と役割

`Output/` フォルダは、このプロジェクトの心臓部です。学習の進捗や解析結果、実践の記録はすべてこのディレクトリ内で体系的に管理されます。

| フォルダ | 役割 | 使い方・保存されるもの |
|---------|------|----------------------|
| **`00_index/`** | 🗺 **総合インデックス** | ECC全体の構造（48 agents, 182 skills, 68 commands 等）を俯瞰するための索引。**「まずここを読む」** エントリポイントです。 |
| **`01_analysis/`** | 🔬 **深掘り解析メモ** | 特定のエージェントのプロンプト構造や、スキルのワークフローなどをAIと共に解剖し、詳細に解説したドキュメント。 |
| **`02_notes/`** | 📓 **読書ノート・考察** | ECCのドキュメントやコードを読んで得た、人間（自分）の気づきや考察をまとめるノート。 |
| **`03_glossary/`** | 辞書 **用語集** | ハーネスエンジニアリング特有の概念や用語（例: "Context Compaction", "PreToolUse" 等）の定義集。 |
| **`04_patterns/`** | 🧩 **設計パターン集** | ECCから抽出した、再利用可能なプロンプトパターンやアーキテクチャパターンのカタログ。 |
| **`05_practice/`** | 💻 **実践・作業場** | 学んだ概念を使って、実際にプロンプトを書いたり、簡単なフックやスキルを再実装してみるサンドボックス。 |
| **`06_questions/`** | ❓ **疑問リスト** | 解析中に生じた未解決の疑問、あるいは解決済みのQAログ。 |
| **`07_comparisons/`** | ⚖️ **比較分析** | 他のAIコーディングツール（Cursor, Gemini等）や、従来の手法とECCの思想・機能の比較。 |
| **`08_roadmap/`** | 🗺️ **学習ロードマップ** | プロジェクトの進捗管理、次に何を学ぶかのタスクリスト。 |
| **`09_logs/`** | 📝 **学習ログ** | 日々の学習の記録（日報・週報）、セッションの振り返り。 |

## 🚀 学習の進め方

1. **インデックスの把握**: まずは `Output/00_index/00_INDEX.md` を起点に、ECCがどのような要素（agents, commands, skills, hooks, rules）で構成されているかの全体像を掴みます。
2. **気になるところの深掘り**: 興味を持ったagents（例: `planner` や `code-reviewer`）のプロンプトを `ECC/` から読み込み、`Output/01_analysis/` に解析結果をまとめます。
3. **パターンの抽出**: 優れたプロンプトの書き方やツールのオーケストレーション手法を発見したら、`Output/04_patterns/` に記録します。
4. **実践**: 抽出したパターンを使って、`Output/05_practice/` で自分なりの小さなagent, command, skill, hook, rule を自作してみます。

## 🤖 AIアシスタント（Claude）との連携

このプロジェクトでは、AIアシスタントに以下のスラッシュコマンドを使って作業を依頼できます。

- `/analyze <パス>`: ECCの特定のファイルを解析し、`01_analysis` に詳細なレポートを作成させる。
- `/note <トピック>`: 議論した内容や気づきを `02_notes` に保存させる。
- `/pattern <パターン名>`: 有用な設計パターンを `04_patterns` に抽出・整理させる。
- `/question <疑問>`: 分からないことを `06_questions` にストックさせる。

（※ 詳細は `CLAUDE.md` を参照してください）
